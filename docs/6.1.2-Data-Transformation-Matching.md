# Transformation et correspondance des données


## Objectif et périmètre

Ce document explique comment les données brutes de films provenant des APIs partenaires sont transformées en entités structurées puis mises en correspondance avec des films dans le backend Patrimony. 
Cela inclut l'extraction des métadonnées, la normalisation des structures de données, et la logique de déduplication qui relie plusieurs sources partenaires à une seule entité `Movie`.

## Vue d'ensemble

Le processus de transformation et de correspondance des données se déroule en trois étapes :

1. **Extraction** : les réponses API ou les fichiers spécifiques à chaque partenaire sont parsées et normalisées
2. **Transformation** : les données sont converties en entités `SourceMovie` avec une structure standardisée
3. **Correspondance** : les entités `SourceMovie` sont liées à des entités `Movie` canoniques via les IDs externes

Cette séparation permet au système de conserver les données spécifiques à chaque partenaire tout en construisant un catalogue unifié et dédupliqué.

## Extraction des données depuis les APIs partenaires

Chaque importer partenaire reçoit les données dans des formats différents et en extrait les champs pertinents. La logique d'extraction est implémentée dans les classes d'importers spécifiques à chaque partenaire.

## Création de l'entité SourceMovie

Après l'extraction, les données sont stockées dans une entité `SourceMovie`. Il s'agit d'un enregistrement spécifique au partenaire qui préserve les données sources originales.

### Rôle du SourceMovieManager

La méthode `SourceMovieManager.findOrCreate()` gère la création des `SourceMovie`.

### Propriétés du SourceMovie

Une fois créée, l'entité `SourceMovie` est alimentée avec les métadonnées extraites :

| Propriété | Type de données | Exemple d'appel setter |
|---|---|---|
| Directors | Tableau | `$sourceMovie->setDirectors($directors)` |
| Casting | Tableau | `$sourceMovie->setCasting($casting)` |
| Synopsis | Chaîne | `$sourceMovie->setSynopsis($synopsis)` |
| Duration | Entier | `$sourceMovie->setDuration($duration)` |
| Poster | Chaîne (URL) | `$sourceMovie->setPoster($poster)` |
| Genres | Tableau | `$sourceMovie->setGenres($genres)` |
| Public | Chaîne | `$sourceMovie->setPublic('12')` |


## Collecte des IDs externes

Les IDs externes sont la clé pour faire correspondre les entités `SourceMovie` aux enregistrements `Movie` canoniques. Chaque partenaire peut fournir des identifiants externes différents.

### Pattern d'extraction des IDs

Les importers construisent un tableau `$ids` contenant tous les identifiants externes disponibles.

### Types d'IDs externes supportés

L'entité `Movie` supporte les champs d'IDs externes suivants :

| Champ | Rôle | Partenaires fournisseurs |
|---|---|---|
| `cncId` | ID de la base de référence CNC | Import CSV CNC |
| `arteId` | ID interne Arte TV | Arte TV |
| `canalVodId` | ID Canal VOD | Canal VOD/Replay |
| `orangeVodId` | ID Orange VOD | Orange VOD |
| `allocineId` | ID film Allociné | France TV, autres |
| `laCinetekId` | ID LaCinetek | LaCinetek |
| `franceTvId` | ID programme France TV | France TV |
| `tf1Id` | ID programme TF1 | TF1 |
| `imdbId` | ID film IMDB | Plusieurs partenaires |
| `plurimediaId` | ID broadcast Plurimedia | France TV, TF1 |
| `isanId` | Identifiant ISAN | CNC, autres |


## Logique de correspondance des films

Le processus de correspondance détermine si un `SourceMovie` doit être lié à un `Movie` existant ou si un nouveau `Movie` doit être créé.

### Service MovieFetcher

La méthode `MovieFetcher.fetchByIds()` effectue la correspondance : elle interroge le `MovieRepository` pour trouver toute entité `Movie` ayant au moins un champ d'ID externe correspondant.

### Création via MovieManager

Lorsqu'aucun `Movie` correspondant n'existe, `MovieManager.create()` génère un nouvel enregistrement :

1. Extraire les métadonnées principales du `SourceMovie` (titre, année de production)
2. Générer un code unique pour le film
3. Définir le flag `hasAd` à `true` (tous les films importés ont une audiodescription)
4. Copier le synopsis, le poster et la durée depuis le `SourceMovie`
5. Définir le champ d'ID externe approprié selon le partenaire source

### Correspondance conditionnelle

L'option `--create-movies` contrôle si la correspondance est effectuée :

- **Désactivée** : les entités `SourceMovie` et `Solution` sont créées mais non liées aux enregistrements `Movie` canoniques
- **Activée** : la correspondance et la liaison complètes sont effectuées, construisant le catalogue unifié

## Relations entre entités lors de la correspondance

Le processus de correspondance établit des relations bidirectionnelles entre les entités :

- **SourceMovie → Movie** : chaque `SourceMovie` est lié à zéro ou un `Movie`
- **Solution → Movie** : chaque `Solution` est liée à zéro ou un `Movie`
- **Solution → SourceMovie** : chaque `Solution` référence le `SourceMovie` depuis lequel elle a été créée
- **Movie → SourceMovies** : un `Movie` agrège plusieurs enregistrements `SourceMovie` de différents partenaires
- **Movie → Solutions** : un `Movie` possède plusieurs enregistrements `Solution` montrant sa disponibilité chez les partenaires

## Services et managers clés

Le processus de transformation et de correspondance s'appuie sur plusieurs classes de services :

| Service | Responsabilité | Méthodes clés |
|---|---|---|
| `SourceMovieManager` | Créer/récupérer les entités `SourceMovie` | `findOrCreate()` |
| `MovieManager` | Créer les entités `Movie` | `create()` |
| `MovieFetcher` | Faire correspondre un `SourceMovie` à un `Movie` existant | `fetchByIds()` |
| `SolutionManager` | Créer/mettre à jour les entités `Solution` | `createOrUpdate()` |
| `MovieRepository` | Interroger les entités `Movie` | `findByIds()` |
| `SourceMovieRepository` | Interroger les entités `SourceMovie` | `findOneBy()` |

### SolutionManager

La méthode `SolutionManager.createOrUpdate()` crée les enregistrements de disponibilité en associant un `SourceMovie`, un partenaire, une offre, une URL de visionnage et les dates de début/fin de droits.

## Exécution des commandes d'import

Le processus de transformation et de correspondance est déclenché via des commandes console Symfony :

| Commande | Source d'import | Option create movies |
|---|---|---|
| `ad:import:francetv-api` | API France TV | `--create-movies=true` |
| `ad:import:tf1-api` | API TF1 | `--create-movies=true` |
| `ad:import:artetv-api` | API Arte TV | `--create-movies=true` |
| `ad:import:canalvod-api` | API Canal VOD | `--create-movies=true` |
| `ad:import:canalreplay-api` | API Canal Replay | `--create-movies=true` |
| `ad:import:lacinetek-api` | API LaCinetek | `--create-movies=true` |
| `ad:import:orangevod-csv` | CSV Orange VOD | `--create-movies=true` |

Tous les imports sont exécutées quotidiennement par le cron à 01h05 via la cible `cron-import` du Makefile.

Toutes les commandes Make sont listées sur la page [Liste des commandes Make](7-Commands-Makefile.md).
