# Moteur de recherche et système de requêtage

Ce document explique le service `MovieSearchManager` et la façon dont il construit les requêtes Elasticsearch via le module Search API. 
Le manager gère la construction des requêtes, l'application des filtres et la pagination pour la fonctionnalité de recherche de films.

Pour les informations sur les formulaires de recherche et les interactions AJAX, voir [Formulaire de recherche et filtres](5.4.1-Search-Fom-Filters.md). 
Pour la configuration de Search API, voir [Configuration Search API (3.4.3)](https://deepwiki.com/betagouv/audiodescription/3.4.3-search-api-configuration).

## Vue d'ensemble du service

Le `MovieSearchManager` est un service dédié responsable de l'abstraction de la construction des requêtes Elasticsearch. 
Il fournit une interface propre entre les contrôleurs/formulaires et l'infrastructure Search API sous-jacente.

### Définition du service

| Propriété | Valeur |
|---|---|
| ID de service | `audiodescription.manager.movie_search` |
| Classe | `Drupal\audiodescription\Manager\MovieSearchManager` |
| Localisation | `src/Manager/MovieSearchManager.php` |
| Dépendances | `EntityTypeManagerInterface` |

Le service est instancié avec l'entity type manager et charge l'index Search API `movies` lors de sa construction.

## Méthodes de construction des requêtes

### Méthode principale : queryMovies()

La méthode `queryMovies()` est l'interface principale pour exécuter les requêtes de recherche. 
Elle retourne le nombre total de résultats, le nombre de pages et les entités chargées.

**Valeur de retour :** `[int $total, int $pagesCount, array $entities]`

## Logique d'application des filtres

### Filtre recherche textuelle

La recherche textuelle utilise la méthode `keys()` de Search API, qui effectue une recherche plein texte sur les champs indexés.

### Filtre genre (logique OR)

Plusieurs genres sont combinés avec une logique OR, retournant les films correspondant à *n'importe quel* genre sélectionné.

### Filtre partenaire (logique OR)

Plusieurs partenaires sont combinés avec une logique OR, retournant les films disponibles sur *n'importe quelle* plateforme sélectionnée.

### Filtre films gratuits

Le filtre films gratuits est le plus complexe. 
Il nécessite une requête SQL pour identifier les partenaires avec des offres gratuites dans leurs fenêtres de droits.

## Chargement des entités et traitement des résultats

### Extraction des IDs

Search API retourne les IDs d'entités au format `entity:node/{nid}:{langue}`. 
Le manager les parse pour extraire les IDs de nœuds :

1. `explode('/', $entity_id)[1]` extrait `{nid}:{langue}`
2. `explode(':', $id)[0]` extrait uniquement le `{nid}`

### Chargement des entités

Les IDs de nœuds sont chargés en lot via l'entity type manager.

## Comptage des films avec audiodescription

La méthode `countAdMovies()` fournit un comptage séparé de tous les films avec audiodescription (avec les mêmes filtres).

> **Note :** le comptage AD applique les mêmes filtres de texte et de genre que la requête principale, mais ajoute une condition supplémentaire `field_has_ad = 1`.

## Construction de la pagination

La méthode `buildPagination()` construit une structure de pagination compatible avec le pattern de pagination DSFR.

### Constante de pagination

| Constante | Valeur | Rôle |
|---|---|---|
| `PAGINATION_SIZE` | 2 | Nombre de liens de page avant/après la page courante |

### Structure de la pagination

Le tableau de pagination retourné contient les éléments `first`, `prev`, `pages[]`, `next` et `last`, chacun avec son URL et son titre.

### Construction des URLs

Toutes les URLs de pagination sont construites via `buildUrl()` qui préserve tous les paramètres de recherche. 
La méthode utilise `Url::fromRoute()` avec la route `audiodescription.movie_search` et inclut tous les paramètres de filtres depuis `MovieSearchParametersBag::filtersToArray()`.

## Points d'intégration

### Intégration dans le contrôleur

Le `MovieSearchController` utilise le manager pour les chargements initiaux de page :

1. Créer `MovieSearchParametersBag` depuis la requête
2. Calculer l'offset depuis le numéro de page
3. Appeler `queryMovies()` avec offset et taille de page
4. Restituer les entités via le view builder

### Intégration dans le formulaire

Le `FiltersMovieSearchForm` utilise le manager pour les mises à jour AJAX :

1. Mettre à jour les paramètres de requête depuis le form state
2. Créer `MovieSearchParametersBag` depuis la requête modifiée
3. Appeler `queryMovies()` avec les paramètres
4. Construire la réponse AJAX avec `ReplaceCommand`
5. Mettre à jour l'URL avec `SearchAjaxUrlUpdateCommand`

### Constantes de taille de page

La constante `PAGE_SIZE` est définie indépendamment dans le contrôleur et dans le formulaire. 
Les deux doivent rester synchronisées.

## Gestion des paramètres de recherche

### Propriétés du MovieSearchParametersBag

| Propriété | Type | Rôle |
|---|---|---|
| `search` | `string` | Requête de recherche plein texte |
| `page` | `int` | Numéro de page courant (base 1) |
| `withAd` | `bool` | Filtre pour l'audiodescription (déprécié) |
| `genre` | `array` | Tableau d'IDs de taxonomie genre |
| `partner` | `array` | Tableau d'IDs de taxonomie partenaire |
| `isFree` | `bool` | Filtre pour les films gratuits uniquement |

### Méthode factory

La méthode statique `createFromRequest()` extrait les paramètres depuis l'objet Request Symfony :

- Utilise `$request->query->get()` pour les valeurs scalaires
- Utilise `$request->query->getIterator()` pour les valeurs de type tableau
- Fournit des valeurs par défaut pour tous les paramètres
- Garantit que la page est au minimum à 1

### Sérialisation pour les URLs

La méthode `filtersToArray()` convertit les paramètres en tableau pour la construction d'URLs :

- Inclut toujours le paramètre `search`
- Inclut conditionnellement les filtres uniquement s'ils ont des valeurs
- Omet les paramètres faux/vides pour garder les URLs propres

### Détection de l'état vide

La méthode `isEmptyParametersBag()` vérifie si des filtres sont actifs.

## Considérations de performance

### Optimisations de requête

Le manager implémente plusieurs optimisations :

1. **Pagination** : utilise `range()` pour limiter les résultats et éviter les grands ensembles
2. **Chargement en lot** : charge les entités en un seul appel `loadMultiple()` plutôt qu'individuellement 
3. **Comptage AD séparé** : la requête de comptage AD est indépendante, permettant des stratégies de cache 

### Requête SQL brute pour les films gratuits

Le filtre films gratuits utilise une requête SQL brute pour des raisons de performance. 
Cela contourne l'API des entités Drupal pour joindre efficacement à travers la hiérarchie de paragraphes et filtrer par contraintes de dates.

**Compromis :**

- Exécution rapide avec une logique JOIN complexe
- Contourne les vérifications d'accès aux entités
- Nécessite une maintenance si la structure des paragraphes change
