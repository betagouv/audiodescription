# Navigation et fils d'Ariane

Ce document décrit les systèmes de navigation dans le frontend Drupal, incluant :
* les fils d'ariane 
* le header
* le footer

L'application Drupal implémente les composants de navigation du Système de Design de l'État français (DSFR).

Pour les informations détaillées sur les builders individuels et leur logique de correspondance de routes, voir [Builders de fil d'Ariane (3.6.1)](https://deepwiki.com/betagouv/audiodescription/3.6.1-breadcrumb-builders). 
Pour la documentation sur les patterns de templates header et footer, voir [Patterns header & footer (3.6.2)](https://deepwiki.com/betagouv/audiodescription/3.6.2-header-and-footer-patterns).

---

## Vue d'ensemble du système de fil d'Ariane

> **Accessibilité** : Suite à des utilisateurs qualitatifs, les fils d'ariane ne sont plus affichés dans l'application.

L'application implémente l'interface `BreadcrumbBuilderInterface` de Drupal pour fournir des fils d'Ariane contextuels. 
7 builders spécialisés gèrent différents types de routes, chacun implémentant une logique personnalisée basée sur le type de contenu, le vocabulaire de taxonomie ou le nom de route.

Tous les builders suivent un pattern commun :

1. Implémenter `applies(RouteMatchInterface $route_match)` pour déterminer si le builder gère la route courante
2. Implémenter `build(RouteMatchInterface $route_match)` pour construire le fil d'Ariane
3. Ajouter des métadonnées de cache pour contrôler le comportement de mise en cache

Les builders sont enregistrés comme services taggués dans Drupal, permettant au gestionnaire de fil d'Ariane de les découvrir et les prioriser automatiquement.

## Récapitulatif des builders de fil d'Ariane

Les 7 builders et leurs responsabilités :

| Classe du builder | Type de route/entité | Pattern du fil d'Ariane                                                                            | Stratégie de cache |
|---|---|----------------------------------------------------------------------------------------------------|---|
| `MovieBreadcrumbBuilder` | `node` de type `movie` | Accueil > [Titre de la vue Genres] > [Vue de recherche filtrée sur le 1er genre] > [Titre du film] | max-age: 0 |
| `SearchBreadcrumbBuilder` | Route `audiodescription.movie_search` | Accueil > Résultats de recherche                                                                   | max-age: 0 |
| `CollectionBreadcrumbBuilder` | `taxonomy_term` bundle `collection` | Accueil > [Nom de la collection]                                                                   | max-age: 0, contexte: url |
| `NodeBreadcrumbBuilder` | `node` de type `article` ou `page` | Accueil > [Chemin du menu] > [Titre de la page courante]                                           | max-age: 0, contexte: url.path, url |
| `DirectorBreadcrumbBuilder` | `taxonomy_term` bundle `director` | Accueil > [Nom du réalisateur]                                                                     | Contexte: url |
| `GenreBreadcrumbBuilder` | `taxonomy_term` bundle `genre` | Accueil > [Titre de la vue Genres] > [Nom du genre]                                                | Contexte: url |
| `ViewBreadcrumbBuilder` | Routes avec paramètres `view_id` et `display_id` | Accueil > [Chemin du menu] > [Titre de la vue] **OU** Accueil > [Titre de la vue]                                   | Contexte: url |

Tous les builders retournent le fil d'Ariane en commençant par un lien "Accueil" créé via `Link::createFromRoute(t('Home'), '<front>')`.

## Construction du fil d'Ariane des films

Le `MovieBreadcrumbBuilder` implémente une logique spécialisée pour construire les fils d'Ariane selon les relations de genre d'un film :

1. Extrait le premier genre depuis `field_genres`
2. Charge la vue des genres pour récupérer son titre
3. Construit une URL de recherche avec le filtre genre
4. Ajoute le lien vers le terme genre

Si aucun genre n'existe, seul le lien d'accueil est inclus.

## Intégration du chemin de menu

Les builders `NodeBreadcrumbBuilder` et `ViewBreadcrumbBuilder` s'intègrent avec le menu principal de Drupal pour construire les fils d'Ariane selon la hiérarchie de menu. 

Le processus est le suivant :

1. Chargement des paramètres de l'arbre du menu principal
2. Itération sur le chemin actif en ordre inverse
3. Extraction des UUIDs des liens de menu
4. Chargement des entités `menu_link_content` correspondantes
5. Construction des liens du fil d'Ariane depuis leurs titres et URLs

Si aucun chemin actif n'existe, un titre de repli est utilisé (titre du nœud ou de la vue).

## Structure de navigation du template de page

Le template de page principal intègre plusieurs éléments de navigation via les patterns DSFR :

- **Skiplinks** (liens d'évitement) : ce sont des raccourcis de navigation clavier vers les sections principales
- **Header** : l'en-tête via le pattern DSFR
- **Footer** : le pied de page via le pattern DSFR
- **Zone de contenu principal** : le contenu principal de la page entouré d'une balise `<main>`

Tous les composants suivent la structure et les conventions de nommage des classes DSFR.

## Structure du header

## Style du fil d'Ariane

Le composant fil d'Ariane DSFR reçoit des surcharges de style personnalisées pour ajuster l'espacement et la présentation :

- Marge supérieure : `1.5rem`
- Marge inférieure : `1rem` (avec `!important` pour surcharger les valeurs par défaut DSFR)
- Marges horizontales : `0` pour s'aligner avec la grille du conteneur

Voir la documentation du DSFR : [Fil d'ariane (breadcrumb)](https://www.systeme-de-design.gouv.fr/version-courante/fr/composants/fil-d-ariane)

## Stratégie de gestion du cache

Les builders implémentent différentes stratégies de cache selon leurs besoins contextuels :

* `setCacheMaxAge(0)` (pas de cache) lorsque le contenu change fréquemment ou qu'il existe des relations dynamiques
* `addCacheContexts(['url'])` lorsque les paramètres d'URL affectent la structure du fil d'Ariane
* `addCacheContexts(['url.path'])` lorsque la construction du chemin de menu dépendante du path

Les builders Movie et Search désactivent entièrement le cache pour garantir que les fils d'Ariane reflètent l'état courant du contenu. 
Les autres builders utilisent des contextes de cache pour varier les fils d'Ariane mis en cache selon l'URL ou le path, permettant un certain niveau de cache tout en maintenant la cohérence.
