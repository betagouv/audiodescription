# Développement Frontend

Ce document couvre le développement frontend du Portail de l'audiodescription, notamment le thème Drupal personnalisé (`ad_theme`), l'architecture SCSS, les composants JavaScript, les templates Twig et la compilation des assets. 
Pour le développement PHP backend (services et importeurs), voir [Développement Backend](8.2.0-Backend-Developement.md). 
Pour l'architecture générale du système de thème et ses hooks, voir [Thème](5.2.0-Theme.md).

## Périmètre et objectifs

Le développement frontend couvre :

- Le **thème Drupal personnalisé** basé sur le Système de Design de l'État Français (DSFR)
- L'**architecture SCSS modulaire** pour le style des composants
- Les **composants JavaScript** utilisant Lit et du JS vanilla pour les fonctionnalités interactives
- La **personnalisation des templates Twig** pour le rendu du contenu
- Le **preprocessing du thème** pour la transformation des données
- La **compilation des assets** depuis les sources vers des bundles prêts pour la production

## Vue d'ensemble de l'architecture du thème

Le thème personnalisé `ad_theme` se trouve dans `drupal/web/themes/ad_theme/` et suit une architecture orientée composants, intégrant les patterns DSFR avec des fonctionnalités spécifiques au Portail de l'audiodescription.

### Structure des répertoires

| Chemin | Rôle |
|---|---|
| `ad_theme.theme` | Implémentations des hooks de thème et fonctions de preprocessing |
| `templates/` | Surcharges de templates Twig organisées par type d'entité |
| `src/scss/` | Fichiers SCSS source organisés par modules et thèmes |
| `src/components/` | Composants JavaScript (Lit) |
| `src/main.js` | Point d'entrée JavaScript |
| `dist/` | Assets CSS et JS compilés |
| `templates/pattern/` | Templates de patterns DSFR (en-tête, pied de page) |

## Organisation SCSS et styles

L'architecture SCSS suit une approche modulaire avec des fichiers séparés pour les composants, les thèmes et les utilitaires.

### Structure des modules

Les modules SCSS sont importés dans `src/scss/modules/_modules.scss`. 
Les thèmes (clair, sombre, DSFR) sont réunis dans `src/scss/themes/_themes.scss`.

### Variantes de thème

Le thème supporte les modes clair et sombre via le système de thèmes du DSFR :

**Thème clair** (`_light.scss`) :
- Ajustement du contraste des badges violet glycine pour la lisibilité

**Thème sombre** (`_dark.scss`) :
- Hérite pour l'instant des valeurs par défaut du thème sombre DSFR

## Composants JavaScript

Le thème combine des web components Lit et du JavaScript vanilla pour les fonctionnalités interactives.

### Architecture des composants

Le point d'entrée `src/main.js` importe tous les composants et les styles globaux. Les composants Lit se trouvent dans `src/components/`.

### Web Components Lit

#### Composant PartnersSelect

Le composant `PartnersSelect` utilise Lit et implémente un pattern manager pour coordonner le comportement entre instances.

**Pattern Manager** :
- Gère l'enregistrement et la désinscription des instances de composant
- Permet une communication coordonnée entre les composants `<partners-select>`

**Définition du composant** :
- Désactive le Shadow DOM pour la compatibilité avec les styles DSFR
- S'enregistre/se désenregistre auprès du manager global lors du `connect`/`disconnect`
- Élément personnalisé : `<partners-select>`

#### Composant RichSelect

Similaire à `PartnersSelect`, ce composant fournit une fonctionnalité de sélection multiple améliorée utilisée dans les filtres de recherche. 
Il est importé dans `src/main.js`.

### Point d'entrée JavaScript

`src/main.js` importe tous les composants et styles globaux :

```javascript

```

## Processus de build et compilation des assets

Le thème utilise un système de build pour compiler le SCSS en CSS et bundler le JavaScript.

### Workflow de développement

1. Modifier les fichiers source dans `src/scss/` ou `src/components/`
2. Lancer la commande de build (script npm ou équivalent)
3. Les assets compilés sont générés dans `dist/`
4. Drupal charge les assets depuis `dist/` en production

### Chargement des assets

Les assets compilés sont déclarés dans le fichier `.info.yml` ou `.libraries.yml` du thème et chargés par le système de bibliothèques d'assets de Drupal.

Le CSS compilé final (`dist/styles.css`) inclut :

- Tous les styles du framework DSFR
- Les styles des composants personnalisés
- Les surcharges du thème
- Les classes utilitaires
