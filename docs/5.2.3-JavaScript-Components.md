# Composants JavaScript

Ce document couvre les composants JavaScript personnalisés utilisés dans le thème Drupal. 
Le système utilise un mélange de web components modernes (Lit) et de behaviors Drupal traditionnels pour enrichir les interactions de formulaires et les fonctionnalités de recherche.

Pour l'intégration JavaScript côté backend et la compilation des assets, voir [Développement frontend (5.1)](https://deepwiki.com/betagouv/audiodescription/5.1-frontend-development). 
Pour l'implémentation du formulaire de recherche qui utilise ces composants, voir [Formulaire de recherche & filtres (3.4.1)](https://deepwiki.com/betagouv/audiodescription/3.4.1-search-form-and-filters).

---

## Vue d'ensemble

Le thème comprend 6 systèmes JavaScript principaux :

| Composant | Technologie | Rôle                                                                |
|---|---|---------------------------------------------------------------------|
| `RichSelect` | Web Component Lit | Dropdown multi-sélection enrichi avec recherche et filtrage         |
| `PartnersSelect` | Web Component Lit | Placeholder pour une future fonctionnalité d'abonnement partenaire  |
| `searchAjaxUpdateLive` | Behavior Drupal | Annonces d'accessibilité lors des mises à jour AJAX de la recherche |
| `searchAjaxUpdateUrl` | Behavior Drupal | Change l'URL de la page lors des mises à jour AJAX de la recherche  |
| `backButton` | Behavior Drupal | Configure le bouton de retour à la page précédente                  |
| `iframeResponsive` | Behavior Drupal | Change la taille de l'iframe si la fenêtre change de taille         |

## Architecture des composants

Le système de build compile les composants Lit dans un bundle unique. 
Les fonctions de preprocessing PHP préparent les attributs de données qui hydratent les web components dans le navigateur.

## Composant RichSelect

La classe `RichSelect` est un web component personnalisé qui étend l'élément select standard de Drupal avec une capacité multi-sélection, une recherche live et une navigation au clavier. 
Il utilise le framework Lit pour le rendu réactif et Fuse.js pour la recherche approximative (fuzzy search).

### Propriétés réactives

Le composant définit plusieurs propriétés réactives qui déclenchent un re-rendu lorsqu'elles sont modifiées :

| Propriété | Type | Rôle |
|---|---|---|
| `title` | Chaîne | Label principal de l'élément select |
| `plural_title` | Chaîne | Forme plurielle pour l'affichage du nombre de sélections |
| `singular_title` | Chaîne | Forme singulière pour une sélection unique |
| `is_female` | Booléen | Genre grammatical français pour les labels |
| `visible` | Booléen | Contrôle l'état de visibilité du dropdown |
| `options` | Chaîne | Paires clé-valeur des options disponibles encodées en JSON |
| `selected` | Chaîne | Tableau des clés sélectionnées encodé en JSON |
| `selectedOptions` | Tableau | État interne des clés d'options sélectionnées |
| `currentOptions` | Tableau | Options filtrées affichées dans le dropdown |

Ces propriétés sont alimentées depuis PHP via la fonction de preprocessing.

### Configuration du Shadow DOM

Le composant désactive explicitement le Shadow DOM en surchargeant `createRenderRoot()`. 
Cela permet au composant d'hériter des styles DSFR du document parent, garantissant une cohérence visuelle avec le système de design de l'État.

### Pattern Manager

L'instance globale `RichSelectManager` garantit qu'un seul dropdown est ouvert à la fois. 
Lorsqu'un utilisateur ouvre un dropdown, le manager ferme tous les autres, évitant les conflits d'interface.

### Génération des labels de bouton

Le composant génère dynamiquement des labels grammaticalement corrects en français :

- 0 sélectionné : "Sélectionner un genre" / "Sélectionner une plateforme"
- 1 sélectionné : "1 genre sélectionné" / "1 plateforme sélectionnée"
- Plusieurs : "3 genres sélectionnés" / "5 plateformes sélectionnées"

La propriété `is_female` contrôle l'accord de genre en français.

### Rendu produit

La méthode `render()` produit une structure fieldset conforme au DSFR. 
La visibilité du dropdown est contrôlée par la classe `ad-dropdown--visible`, basculée par la propriété `visible`. 
Chaque case à cocher maintient son propre état `checked` en fonction de `selectedOptions`.

## Behavior Drupal : mise à jour live AJAX de la recherche

Le behavior Drupal `searchAjaxUpdateLive` fournit des annonces d'accessibilité lorsque les résultats de recherche sont mis à jour via AJAX.

## Behavior Drupal : mise à jour de l'URL de la page de recherche avec Ajax

Le behavior Drupal `searchAjaxUpdateUrl` permet de modifier l'URL de la page de recherche quand des filtres sont ajoutés / modifiés / supprimés.

## Behavior Drupal : backButton

Le behavior Drupal `backButton` permet de gérer le comportement du bouton "Retour à la page précédente".

## Behavior Drupal : iframeResponsive

Le behavior Drupal `iframeResponsive` recalcule les dimensions de l'iframe quand la fenêtre du navigateur est redimensionnée.
