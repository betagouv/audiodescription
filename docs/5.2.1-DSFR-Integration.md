# Intégration DSFR

Ce document explique comment le Système de Design de l'État Français (DSFR) est intégré dans le thème Drupal personnalisé `ad_theme`. 
Il couvre l'utilisation des composants DSFR, les propriétés CSS personnalisées, la création de templates via des patterns, et les surcharges spécifiques au thème.

Pour les informations sur le preprocessing du thème et la préparation des données, voir [Preprocessing du thème (3.2.2)](https://deepwiki.com/betagouv/audiodescription/3.2.2-theme-preprocessing). 
Pour les composants JavaScript qui étendent les fonctionnalités DSFR, voir [Composants JavaScript (3.2.3)](https://deepwiki.com/betagouv/audiodescription/3.2.3-javascript-components).

## Vue d'ensemble

Le DSFR est le Système de Design de l'État français pour les services numériques du gouvernement français, fournissant des composants standardisés, une typographie, des couleurs... 

Le thème `ad_theme` intègre le DSFR en se basant sur le thème Drupal `ui_suite_dsfr`.

Il implémente le DSFR en : 
- Utilisant les classes CSS DSFR pour tous les composants d'interface
- Exploitant les propriétés CSS personnalisées du DSFR (design tokens) pour le thème
- Implémentant les patterns DSFR via des inclusions de patterns Twig
- Étendant les composants DSFR avec des modules SCSS personnalisés
- Supportant le système de thème clair/sombre du DSFR

## Propriétés CSS personnalisées (design tokens)

Le DSFR fournit des propriétés CSS personnalisées (variables CSS) pour un thème cohérent. 
Le thème utilise massivement ces tokens dans ses composants personnalisés.

### Tokens DSFR couramment utilisés

| Catégorie de token | Exemples de variables | Usage |
|---|---|---|
| **Couleurs de fond** | `--background-default-grey`, `--background-alt-blue-france`, `--background-contrast-grey`, `--background-contrast-info` | Fonds de blocs, champs de formulaire, menus déroulants |
| **Couleurs de bordure** | `--border-plain-grey` | Bordures des champs, séparateurs de composants |
| **Couleurs de texte** | `--text-default-grey`, `--text-action-high-blue-france`, `--text-mention-grey`, `--text-active-blue-france` | Typographie, liens, texte secondaire |
| **Couleurs artistiques** | `--artwork-major-blue-france` | Éléments d'accentuation, états de focus |
| **Espacement** | `--text-spacing` | Marges et padding cohérents |

## Classes de composants DSFR

Le thème utilise les classes de composants DSFR dans l'ensemble des templates. Toutes les classes DSFR suivent la convention de nommage `fr-*`.

### Composants DSFR principaux utilisés

| Composant | Classes DSFR | Emplacement template |
|---|---|---|
| **Header** | `fr-header`, `fr-header__body`, `fr-header__brand`, `fr-header__service`, `fr-header__tools`, `fr-header__navbar`, `fr-header__menu` | `pattern-header.html.twig` |
| **Footer** | `fr-footer`, `fr-footer__top`, `fr-footer__body`, `fr-footer__brand`, `fr-footer__content`, `fr-footer__bottom` | `pattern-footer.html.twig` |
| **Logo** | `fr-logo` | Header et footer |
| **Carte** | `fr-card`, `fr-card__header`, `fr-card__body`, `fr-card__content`, `fr-card__desc`, `fr-card__img`, `fr-enlarge-link` | Templates d'affichage des films |
| **Badge** | `fr-badge`, `fr-badge--purple-glycine` | Affichage des genres/tags |
| **Grille** | `fr-grid-row`, `fr-grid-row--gutters`, `fr-col`, `fr-col-12`, `fr-col-md-3` | Mise en page |
| **Bouton** | `fr-btn`, `fr-btn--search`, `fr-btn--menu`, `fr-btn--close`, `fr-btn--secondary` | Actions et navigation |
| **Modal** | `fr-modal` | Recherche et menu mobile |
| **Tuile** | `fr-tile` | Liste des genres |
| **Fil d'Ariane** | `fr-breadcrumb` | Navigation |

## Système de patterns

Le thème implémente les composants DSFR via le système de patterns de Drupal grâce à la fonction Twig `pattern()`. 
Cette approche centralise le balisage DSFR et permet des composants réutilisables.

## Personnalisations et surcharges DSFR

Le thème inclut des surcharges minimales mais nécessaires aux défauts DSFR pour répondre aux exigences spécifiques du projet.
Ce code spécifique est disponible dans `ad_theme/src/scss/themes/`.


## Intégration avec les composants personnalisés

Les composants personnalisés étendent le DSFR en utilisant ses design tokens tout en ajoutant des fonctionnalités spécifiques au projet.

### Exemple : composant Rich Select

Le composant `.audiodesc-rich-select` illustre une bonne intégration DSFR :

**Tokens DSFR utilisés :**
- `--background-contrast-grey` pour le fond du bouton
- `--border-plain-grey` pour les bordures
- `--text-default-grey` pour les couleurs de texte
- `--background-default-grey` pour le panneau dropdown

**Classes DSFR incorporées :**
- `.fr-fieldset` pour la structure du formulaire
- `.fr-fieldset__element` pour les éléments du champ
- `.fr-label` pour les libellés
- `.fr-text` pour le texte descriptif

**Comportement personnalisé ajouté :**
- `.audiodesc-rich-select__btn-control-dropdown` pour le style du bouton personnalisé
- `.audiodesc-rich-select__dropdown` pour le positionnement du dropdown
- `.audiodesc-rich-select__dropdown-options` pour la liste d'options scrollable
