# Cron jobs et automatisation

Cette page documente les tâches planifiées et le système d'automatisation qui maintient les données films fraîches dans le Portail de l'audiodescription. 
Le système utilise un cron quotidien pour importer les données depuis les APIs partenaires et les synchroniser vers le frontend Drupal.
Il utilise aussi un cron hebdomadaire pour générer et envoyer la newsletter aux abonnés.

Pour les détails sur le fonctionnement interne du pipeline d'import et les intégrations APIs partenaires, voir [Importers (synchronisation avec Patrimony)](5.1.2-Importers.md). 
Pour l'infrastructure de déploiement, voir [Configuration Docker](7.1.0-Docker-Setup.md).

## Cron job d'import quotidien

### Configuration de la planification

Le système de production exécute un import quotidien à **01h05** (fuseau Europe/Paris) via cron. 
L'entrée crontab est définie dans `scripts/crontab` :

```
5 1 * * * cd /opt/audiodescription && make cron-import >> /opt/audiodescription/logs/cron-import.log
```

Ce job exécute la cible `cron-import` du Makefile et ajoute la sortie au fichier de log `/opt/audiodescription/logs/cron-import.log`.

### Processus d'import cron

La cible `cron-import` exécute un processus d'import en deux phases.

#### Phase 1 : Import des données partenaires

Le système importe séquentiellement les données films depuis les données partenaires (API ou fichier). 
Chaque commande d'import inclut le flag `--create-movies=true` pour activer la canonicalisation automatique des films. 
Le flag `-T` désactive l'allocation de pseudo-TTY pour l'exécution non-interactive.

Chaque commande est suivie de `|| true` pour éviter que l'échec d'un partenaire n'arrête l'ensemble du pipeline. 
Ainsi, une défaillance (timeout API, problème d'identifiants) n'empêche pas les autres partenaires d'être importés.

#### Phase 2 : Synchronisation Drupal

Après l'import Patrimony, le système synchronise les données vers Drupal via 3 commandes Drush :

- `adia` (alias de `ad:import:patrimony:all`) importe films, réalisateurs, genres et autres entités depuis la base Patrimony vers les nœuds et termes de taxonomie Drupal.
- `adum` (alias de `ad:unpublish-movies`) dépublie les films qui n'ont plus de solution active.
- `cr` reconstruit le cache pour que le nouveau contenu soit visible.

## Cron job d'envoi de la newsletter hebdomadaire

### Configuration de la planification

Le système de production exécute envoie une newsletter tous les lundis à **12h30** (fuseau Europe/Paris) via cron.
L'entrée crontab est définie dans `scripts/crontab` :

```
30 12 * * 1 cd /opt/audiodescription && make pt-send-newsletter > /var/log/audiodescription/cron-newsletter.log 2>&1
```

Ce job exécute la cible `pt-send-newsletter` du Makefile et ajoute la sortie au fichier de log `/opt/audiodescription/logs/cron-newsletter.log`.

## Cibles d'automatisation du Makefile

Le Makefile fournit plusieurs cibles d'automatisation pour différents scénarios d'import. 
Toutes utilisent `docker compose exec` pour exécuter les commandes dans les conteneurs.

Toutes les commandes Make sont détaillées sur la page [Liste des commandes Make](7.4.0-Commands-Makefile.md)

## Commandes console Symfony

L'application Patrimony fournit des commandes console Symfony pour chaque source d'import. 
Ces commandes sont invoquées par les cibles du Makefile.

### Pattern des commandes

Toutes les commandes d'import suivent un pattern cohérent :

```
php bin/console ad:import:<source> [--create-movies=<true|false>]
```

Les types de sources d'import sont définis comme un enum dans `patrimony/src/Enum/ImportSourceType.php` :

| Commande | Valeur enum | Partenaire |
|---|---|---|
| `ad:import:francetv-api` | `FRANCE_TV_API` | France Télévisions |
| `ad:import:tf1-api` | `TF1_API` | TF1+ |
| `ad:import:artetv-api` | `ARTE_TV_API` | Arte TV |
| `ad:import:canalvod-api` | `CANAL_VOD_API` | Canal+ VOD |
| `ad:import:canalreplay-api` | `CANAL_REPLAY_API` | Canal+ Replay |
| `ad:import:lacinetek-api` | `LACINETEK_API` | LaCinetek |
| `ad:import:orangevod-csv` | `ORANGE_VOD_CSV` | Orange VOD |
| `ad:import:cnc-public` | `CNC_CSV` | Données de référence CNC |


## Commandes Drush d'automatisation

Après l'import des données Patrimony, les commandes Drush synchronisent les données vers les entités Drupal.

### Aliases Drush principaux

| Commande | Alias | Rôle |
|---|---|---|
| `drush ad:import:patrimony:all` | `adia` | Importe toutes les entités Patrimony vers Drupal |
| `drush ad:update:metatags` | `adum` | Met à jour les metatags des nœuds films |
| `drush cr` | — | Reconstruit tous les caches Drupal |
