# Formulaire de recherche et filtres

Ce document décrit la classe `FiltersMovieSearchForm` et ses systèmes associés, qui fournissent un filtrage multi-critères pour l'interface de recherche de films. 
Le formulaire permet aux utilisateurs de filtrer les films par genre, plateforme partenaire et disponibilité (gratuit vs payant), avec des mises à jour AJAX qui modifient les résultats sans rechargement de page.

Pour les informations sur la traduction des filtres en requêtes Elasticsearch et l'exécution des résultats, voir [Manager de recherche & construction de requêtes (3.4.2)](https://deepwiki.com/betagouv/audiodescription/3.4.2-search-manager-and-query-building). 
Pour la configuration de Search API et Elasticsearch, voir [Configuration Search API (3.4.3)](https://deepwiki.com/betagouv/audiodescription/3.4.3-search-api-configuration).

## Vue d'ensemble

Le formulaire de filtres est rendu dans la colonne gauche de la page de recherche et fournit 3 types de filtres :

| Type de filtre | Nom du champ | Type de saisie | Source des données |
|---|---|---|---|
| Films gratuits uniquement | `is_free` | Case à cocher | Flag booléen |
| Plateformes partenaires | `partner` | Cases à cocher (rich-select) | Termes de taxonomie partenaire |
| Genres | `genre` | Cases à cocher (rich-select) | Termes de taxonomie genre |

Tous les changements de filtres déclenchent des callbacks AJAX qui mettent à jour simultanément les résultats de recherche, la pagination et les paramètres d'URL sans rechargement complet de la page.

## Architecture du formulaire

`FiltersMovieSearchForm` étend `AbstractMovieSearchForm` et dépend de 3 services injectés. 
Le formulaire travaille étroitement avec `MovieSearchParametersBag` pour gérer l'état des filtres entre les requêtes.

## Configuration des champs du formulaire

### Bouton de réinitialisation

Le bouton de réinitialisation est implémenté comme un élément HTML `<a>` avec le style DSFR. 
Il redirige l'utilisateur vers la page de recherche non filtrée pour effacer toutes les sélections de filtres côté client.

### Case à cocher films gratuits
```
Champ : is_free
Type : checkbox
AJAX : déclenché sur l'événement 'change'
Valeur par défaut : depuis le paramètre de requête 'is_free'
```

### Filtre partenaire (Rich Select)

Le filtre partenaire charge les termes de taxonomie depuis le vocabulaire `partner` et les restitue comme cases à cocher enrichies avec le composant JavaScript rich-select.

```
Champ : partner
Type : checkboxes (multiple)
Options : chargées depuis le stockage taxonomy_term, vocabulaire 'partner'
Propriétés personnalisées :
  - #ad_plugin_type: 'rich-select'
  - #singular_title: 'plateforme'
  - #plural_title: 'plateformes'
  - #is_female: TRUE
  - #is_rich_select: TRUE
```

### Filtre genre (Rich Select)

Similaire au filtre partenaire, mais charge depuis le vocabulaire `genre`.

```
Champ : genre
Type : checkboxes (multiple)
Options : chargées depuis le stockage taxonomy_term, vocabulaire 'genre'
Propriétés personnalisées :
  - #ad_plugin_type: 'rich-select'
  - #singular_title: 'genre'
  - #plural_title: 'genres'
  - #is_female: FALSE
  - #is_rich_select: TRUE
```

> **Note :** les propriétés personnalisées `#singular_title` et `#plural_title` sont utilisées par le composant JavaScript rich-select pour les labels d'accessibilité et d'interface. Voir [Composants JavaScript](5.2.3-JavaScript-Components.md) pour les détails d'implémentation.

## Implémentation du callback AJAX

### Méthode : searchMovies()

Cette méthode gère tous les événements de changement de filtre AJAX. Elle suit six étapes :

* **Étape 1 : Extraire la saisie utilisateur et mettre à jour l'objet Request**. 
Les appels à `array_filter()` suppriment les cases non cochées (qui ont la valeur `0`) des tableaux.

* **Étape 2 : Créer le parameter bag et calculer l'offset**

* **Étape 3 : Exécuter les requêtes de recherche et de comptage**

* **Étape 4 : Restituer les entités film**

* **Étape 5 : Construire la pagination**

* **Étape 6 : Retourner la réponse AJAX**

La `ReplaceCommand` met à jour le `<div id="ajax">` avec les nouveaux résultats. 
La `SearchAjaxUrlUpdateCommand` est une commande AJAX personnalisée qui met à jour l'URL du navigateur pour refléter l'état courant des filtres.


## Gestion des paramètres

### Classe MovieSearchParametersBag

Ce Plain Old PHP Object (POPO) encapsule tous les paramètres de recherche avec leurs valeurs par défaut : `search` (chaîne vide), `page` (1), `withAd` (false), `genre` (tableau vide), `partner` (tableau vide), `isFree` (false).

### Méthode factory : createFromRequest()

Extrait les paramètres depuis un objet `Request`. La méthode `getIterator()` est nécessaire pour accéder aux paramètres de requête de type tableau.

### Méthode : filtersToArray()

Convertit le parameter bag en tableau pour la génération d'URL. N'inclut que les valeurs non vides.

### Méthode : isEmptyParametersBag()

Vérifie si des filtres sont actifs.

## Intégration avec la page de recherche

### Intégration dans le contrôleur

Le `MovieSearchController` instancie le formulaire de filtres et le passe au template . 
Il extrait également les valeurs de filtres courants pour les afficher comme tags.

### Rendu dans le template

Le formulaire de filtres est rendu dans la colonne gauche de la page.

### Conteneur cible AJAX

Les résultats sont encapsulés dans un `<div id="ajax">` qui est remplacé par les réponses AJAX.

## Bibliothèques JavaScript

Le formulaire attache deux bibliothèques JavaScript :

* **Bibliothèque `ajax_filter_update`** gère la commande personnalisée `SearchAjaxUrlUpdateCommand` qui synchronise l'état des filtres avec l'URL du navigateur après les mises à jour AJAX.

* **Bibliothèque `ajax_live_update`** fournit des mises à jour de live region pour les lecteurs d'écran, annonçant les changements du nombre de résultats lorsque les filtres sont appliqués.

Pour les détails d'implémentation du composant rich-select et des autres enrichissements JavaScript, voir [Composants JavaScript](5.2.3-JavaScript-Components.md).

## Comportement de soumission du formulaire

### Méthode : submitForm()

Lorsque le formulaire est soumis via POST traditionnel (sans AJAX), il redirige vers la route de recherche avec les paramètres de requête. 
La méthode `getBaseParameters()` est héritée de `AbstractMovieSearchForm`.

### Action du formulaire

L'action du formulaire inclut un identifiant de fragment `#liste` pour scroller jusqu'à la section des résultats après la soumission.

## Implémentation du filtre films gratuits

Le filtre `is_free` nécessite un traitement spécial car il doit interroger les films ayant des offres `FREE_ACCESS` avec des fenêtres de droits valides.

Lorsque `is_free` est coché, le `MovieSearchManager` exécute une requête SQL complexe pour trouver les partenaires éligibles. 
Cette requête trouve tous les partenaires qui proposent actuellement des films gratuitement (en tenant compte des fenêtres de droits), puis la recherche est filtrée pour n'inclure que les films disponibles chez ces partenaires.

## Constantes

- `PAGE_SIZE` : nombre de résultats par page
- `PAGINATION_SIZE` : nombre de liens de page à afficher avant et après la page courante (ex. avec la valeur `2`, affiche 2 pages avant et 2 pages après)

Ces constantes sont également définies dans `MovieSearchController` et `MovieSearchManager`, assurant la cohérence à travers le système de recherche.

## Résumé

Le `FiltersMovieSearchForm` fournit une interface de filtrage sophistiquée avec :

- **3 types de filtres** : films gratuits (booléen), partenaires (taxonomie), genres (taxonomie)
- **Mises à jour AJAX** : les résultats se rafraîchissent sans rechargement lors des changements de filtres
- **Synchronisation URL** : l'état des filtres est préservé dans les paramètres URL pour les favoris et le partage
- **Enrichissement rich-select** : le composant JavaScript personnalisé améliore l'ergonomie des cases à cocher
- **Accessibilité** : les live regions annoncent les changements de résultats, attributs ARIA sur les boutons de suppression
- **Rendu côté serveur** : les cartes films sont rendues côté serveur et injectées via AJAX
