# Patrimony

---

## Objectif et périmètre

Le backend Patrimony est une application Symfony qui sert à **agréger et nettoyer les données des films** (fournies par les partenaires) pour le Portail de l'audiodescription. 

Il est responsable de :

- La récupération quotidienne des données films depuis les APIs et fichiers fournis par les partenaires
- La création et la mise à jour des films avec leurs métadonnées
- Le stockage des solutions de disponibilité (offres partenaires avec fenêtres de droits)
- La génération de newsletters hebdomadaires avec une sélection de films gratuits (disponibles sur le Portail)
- L'exposition des données films via API pour la synchronisation vers Drupal

## Vue d'ensemble de l'architecture

## Responsabilités principales

### 1. Pipeline d'import des données

Patrimony orchestre les imports quotidiens depuis plusieurs partenaires de contenu. Chaque import suit ce schéma :

1. **Récupération des données** depuis l'API partenaire ou un fichier CSV
2. **Création/mise à jour des entités SourceMovie** pour les données brutes partenaires
3. **Création/mise à jour des entités Solution** pour la disponibilité (partenaire + offre + fenêtres de droits)
4. **Correspondance avec un film** via des IDs externes (IMDb, Allociné, IDs partenaires)
5. **Nettoyage** des solutions expirées et des source movies orphelins

L'`ImporterFactory` crée des instances d'importer selon l'enum `ImportSourceType`.

Plus d'informations sur la page [Pipeline d'import des données](./6.1-Pipeline-Import.md)


### 2. Données

Patrimony maintient une **entité Film** qui agrège les données de plusieurs partenaires sources. Chaque film possède :

- **IDs externes** : champs pour les identifiants spécifiques à chaque partenaire (`franceTvId`, `tf1Id`, `arteId`, `imdbId`, `allocineId`, etc.)
- **Métadonnées canoniques** : titre, synopsis, année de production, réalisateurs, genres, etc.
- **Relations SourceMovie** : liens vers les données brutes des partenaires
- **Relations Solution** : disponibilités temporalisées sur des partenaires spécifiques

Le service `MovieFetcher` effectue la logique de correspondance via la méthode `MovieRepository::findByIds()`, qui interroge l'ensemble des champs d'IDs externes.

### 3. Génération de la newsletter

Le backend Patrimony expose un endpoint à `/newsletter/generator` qui interroge les films et restitue un template HTML pour la newsletter hebdomadaire. Le `NewsletterController` implémente la logique de sélection suivante :

- **Nouveaux films** : films dont les droits ont démarré dans les 7 derniers jours (max 6 : 2 Arte + 1 TF1 + 3 France TV)
- **Films bientôt indisponibles** : films expirant dans 3 à 15 jours (max 7)
- **Films aléatoires** : films gratuits supplémentaires pour compléter les emplacements restants (max 4)

Le contrôleur utilise des méthodes de repository spécialisées : `findNewFreeMovies()`, `findNearEndFreeMovies()`, `findNotSelectedFreeMovies()`, et `countFreeMovies()`.

### 4. Exposition de l'API Rest

Patrimony expose les données films et solutions via les endpoints API Platform (API Rest) :

- `GET /api/v1/movies` : collection de films canoniques avec solutions, genres et réalisateurs intégrés
- `GET /api/v1/solutions` : collection de solutions de disponibilité avec détails partenaire/offre

Ces endpoints utilisent des groupes de sérialisation (`movie:list`, `solution:list`, etc.) pour contrôler les champs exposés.

## Vue d'ensemble du modèle d'entités

La base de données Patrimony utilise un **pattern double entité** avec des entités sources (données brutes partenaires) et des entités patrimony (enregistrements canoniques) :

**Entités clés :**

- **Movie** : enregistrement film canonique avec les IDs externes de tous les partenaires
- **SourceMovie** : données brutes d'un film provenant d'un partenaire spécifique (champs JSON pour les réalisateurs, le casting, les genres)
- **Solution** : disponibilité temporalisée (partenaire + offre + fenêtres de droits + lien)
- **Partner** : plateforme sur laquelle les films sont disponibles (FRANCE_TV, TF1, ARTE, etc.)
- **Offer** : type d'accès (FREE_ACCESS, SVOD, TVOD, etc.)
- **Director** : entité personne liée aux films
- **Genre** : terme taxonomique lié aux films
- **PublicRestriction** : classification par âge (10, 12, 16, 18)
- **ActorMovie** : entité de jointure pour les relations acteur-film-rôle

## Workflow d'import

Le processus d'import suit un schéma cohérent pour tous les partenaires. Voici le déroulé détaillé pour un importer :

**Étapes clés :**

1. **Initialisation Partenaire/Offre** : s'assurer que les entités Partner et Offer existent en base de données
2. **Nettoyage** : marquer tous les films de ce partenaire comme mis à jour, supprimer les solutions/source movies existants
3. **Récupération des données** : requête HTTP vers l'API partenaire ou parsing CSV
4. **Création de SourceMovie** : stockage des données brutes partenaires avec `internalPartnerId`
5. **Extraction des métadonnées** : parsing des réalisateurs, casting, genres, synopsis depuis le format partenaire
6. **Création de Solution** : stockage de la disponibilité avec fenêtres de droits et lien
7. **Correspondance** : utilisation des IDs externes (IMDb, Allociné, IDs partenaires) pour trouver ou créer un Film
8. **Flush et Clear** : persistance en base de données et vidage de l'entity manager pour éviter les fuites mémoire

Les credentials et URLs des APIs partenaires sont stockés dans des variables d'environnement et mappés dans le fichier de configuration .env.
Ces paramètres sont injectés dans les constructeurs des importers via le conteneur d'injection de dépendances de Symfony.


## Système de newsletter

Le système de génération de newsletter interroge la base Patrimony pour sélectionner des films selon leur disponibilité et leurs fenêtres de droits.

**Sections de la newsletter :**

1. **Nouveaux films** (6 au total) : nouvelles additions gratuites récentes, avec priorité à Arte (2) et TF1 (1), le reste depuis France TV
2. **Films bientôt indisponibles** (7) : films expirant dans 3 à 15 jours, triés par `endRights` croissant
3. **Films aléatoires** (4) : sélection aléatoire dans le catalogue gratuit restant
4. **Compteur total** : affichage du nombre total de films gratuits disponibles sur le portail

Le template restitue chaque film avec son titre, ses réalisateurs, son année de production, ses genres, son synopsis, les logos des partenaires et les liens de visionnage avec les dates de fenêtre de droits.

## Points d'intégration

Le backend Patrimony s'interface avec des systèmes externes à trois niveaux :

### 1. APIs partenaires (flux entrant)
Des requêtes HTTP quotidiennes vers les endpoints partenaires récupèrent les catalogues de films. Chaque partenaire a son propre système d'authentification et son propre format de réponse.

### 2. Drupal (flux sortant)
L'application Drupal consomme les données de Patrimony via l'API Rest (API Platform).

### 3. Service newsletter (flux sortant)
Le HTML généré par le service `Service/NewsletterContentGenerator.php` est utilisé pour envoyer directement une newsletter via la commande `php bin/console app:send-weekly-newsletter`.
