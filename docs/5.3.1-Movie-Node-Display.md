# Templates de nœuds film

Ce document décrit les templates Twig utilisés pour afficher les nœuds films dans différents contextes du frontend Drupal. 
Les nœuds films sont affichés via plusieurs variantes de modes d'affichage, chacune optimisée pour des cas d'usage spécifiques : cartes sur la page d'accueil, résultats de recherche, listes générales...

Pour les informations sur l'affichage complet du nœud film, voir [Affichage des offres & partenaires (3.3.2)](https://deepwiki.com/betagouv/audiodescription/3.3.2-offer-and-partner-display). 
Pour la logique de preprocessing du thème, voir [Preprocessing du thème ](5.2.2-Theme-Preprocessing.md).

## Vue d'ensemble des variantes de templates

Le système d'affichage des films fournit quatre variantes de templates principales, chacune répondant à des exigences de présentation différentes :

| Mode d'affichage | Template | Contexte principal                                         |
|---|---|------------------------------------------------------------|
| `card` | `node--movie--card.html.twig` | Collections de la page d'accueil, des pages d'informations |
| `teaser` | `node--movie--teaser.html.twig` | Pages de taxonomie de collections                          |
| `card-result` | `node--movie--card-result.html.twig` | Résultats de recherche (titre h3)                          |
| `card-result-h4` | `node--movie--card-result-h4.html.twig` | Résultats de recherche (titre h4)                          |

Tous les templates suivent la structure du composant carte DSFR avec une mise en page horizontale (`fr-card--horizontal`).

## Pattern commun d'extraction des données

Tous les templates films suivent un pattern cohérent d'extraction des données en début de fichier. La séquence d'extraction est la suivante :

1. **Extraction des réalisateurs** : itération sur `field_directors` pour construire un tableau de noms
2. **Flag audiodescription** : vérification de `field_has_ad.value` et définition du label approprié
3. **Image d'affiche** : préférence pour `field_poster_external.value`, repli sur l'entité media si nécessaire
4. **Construction de la phrase descriptive** : construction d'un tableau de phrases avec réalisateurs et année de production
5. **Traitement du synopsis** : rendu et suppression des balises HTML, troncature à 315 caractères si nécessaire

## Affichage du badge audiodescription

Tous les templates affichent un badge indiquant la disponibilité de l'audiodescription via le composant badge DSFR. 
Le badge apparaît dans la section de début de la carte avec un libellé conditionnel.

### Logique du badge

| Condition | Libellé | Classe badge |
|---|---|---|
| `field_has_ad == true` | "Audiodescription disponible" | `fr-badge fr-badge--purple-glycine` |
| `field_has_ad == false` | "Audiodescription introuvable" | `fr-badge fr-badge--purple-glycine` |

La variante de couleur purple-glycine fait partie de la palette de couleurs du DSFR, offrant une identité visuelle cohérente pour cette fonctionnalité.

## Construction de la phrase descriptive

Chaque template construit un tableau de phrases descriptives qui combine les noms des réalisateurs et l'année de production en texte formaté, affiché sous le titre du film. 
La phrase est rendue avec le filtre `|raw` pour préserver les balises HTML `<strong>` pour l'emphase visuelle.

## Traitement du synopsis

Le champ synopsis contient du contenu HTML qui nécessite un traitement avant l'affichage. 
Les templates appliquent une logique cohérente de rendu et de troncature.

### Étapes de traitement

1. **Rendu et suppression des balises** : le filtre `|render` traite le champ, puis `|striptags` supprime le balisage HTML pour un affichage en texte brut.

2. **Troncature à 315 caractères** : le synopsis est tronqué à 315 caractères avec ajout d'une ellipse.

3. **Affichage conditionnel** :
    - Si le synopsis est présent : affiche la ligne de phrase + synopsis
    - Si le synopsis est vide : affiche chaque partie de la phrase séparément

> **Note :** le template **Card** omet entièrement l'affichage du synopsis, n'affichant que la phrase réalisateur/année.

## Affichage des tags de genres

Les termes de taxonomie genre sont rendus comme composants tag DSFR avec une mise en page conditionnelle selon la quantité.

### Variantes d'implémentation

**Templates Teaser / Card Result** :
- Affichent tous les genres sans limite
- Les genres utilisent `include('ui_suite_dsfr:tag')` pour le rendu du composant DSFR
- Dans la variante card-result, les genres renvoient vers la page de recherche : `/recherche?genre[0]={term.id()}`

**Template Card** :
- Limite l'affichage aux **3 premiers genres** via le filtre `|slice(0, 3)`
- Même approche de rendu par pattern

## Affichage de la disponibilité partenaire

Le template **Card** affiche de manière unique les informations de disponibilité partenaire lorsqu'elles sont présentes, indiquant sur quelle plateforme de streaming le film est disponible et jusqu'à quand.

### Structure des données partenaire

La variable `partner` est transmise avec les propriétés suivantes :

| Propriété | Description | Usage |
|---|---|---|
| `partner.name` | Nom d'affichage du partenaire | Identifiant principal de la plateforme |
| `partner.pronunciation` | Prononciation pour l'attribut ARIA | Accessibilité pour les lecteurs d'écran |
| `partner.end_date` | Date de fin des droits | Notice d'expiration optionnelle |

La variable `partner` est alimentée par la couche de preprocessing du thème (voir [Preprocessing du thème](5.2.2-Theme-Preprocessing.md)) à partir des références de paragraphes `field_offers` du film.

> **Note :** cette fonctionnalité n'apparaît que dans la variante template **Card**, pas dans les templates Teaser ou Card Result.

## Gestion de l'image d'affiche

Tous les templates implémentent un système de repli à deux niveaux pour les images d'affiche, préférant les URLs externes aux entités media.

### Priorité des sources d'image

Le système d'assignation séquentielle garantit que `field_poster_external` écrase toujours la valeur de l'entité media si elle est présente. 
La classe `fr-responsive-img` assure la gestion responsive des images par le DSFR.

> **Note :** Les images utilisent des attributs `alt=""` vides car ce sont des contenus décoratifs, non informatifs.

## Style des cartes

La classe `ad-movie-card` fournit des styles personnalisés pour les composants de carte film au-delà du framework DSFR de base.

### Spécifications de style

**Gestion de l'image :**
- Supprime le ratio d'aspect DSFR par défaut : `aspect-ratio: initial`
- Dimensions fixes sur desktop : `flex: 0 0 200px; width: 200px` pour le header
- Hauteur minimale de la carte : `min-height: 271px` sur desktop (≥768px)

**Typographie :**
- Texte de description : `font-size: .85rem; margin-top: .25rem`
- Tags : `font-size: .85rem; margin-top: .25rem`
- Infos plateforme : `font-size: .85rem; order: 5`

**Comportement responsive :** la propriété `order: 5` sur `.ad-movie-card__platform` garantit que les informations partenaires s'affichent après les autres contenus dans la mise en page flexbox.
