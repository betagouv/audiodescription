# Thème

## Objectif et périmètre

Le thème (`ad_theme`) fournit la couche de présentation visuelle du Portail de l'audiodescription. Il intègre le Système de Design de l'État français (DSFR) avec des composants et styles personnalisés spécifiques à la navigation parmi les films et à l'affichage de la disponibilité partenaire.

Pour des informations complémentaires, voir :

- Logique de rendu du contenu et contrôleurs : [Module Audiodescription](5.1.0-Audiodescription-Module.md)
- Détails de l'intégration du DSFR : [Intégration DSFR (3.2.1)](https://deepwiki.com/betagouv/audiodescription/3.2.1-dsfr-integration)
- Logique de preprocessing du thème : [Preprocessing du thème (3.2.2)](https://deepwiki.com/betagouv/audiodescription/3.2.2-theme-preprocessing)
- Implémentation des composants JavaScript : [Composants JavaScript (3.2.3)](https://deepwiki.com/betagouv/audiodescription/3.2.3-javascript-components)

## Architecture du thème

Le thème personnalisé `ad_theme` est situé dans `drupal/web/themes/ad_theme/` et fournit des surcharges de templates, des fonctions de preprocessing, des modules SCSS et des composants JavaScript pour l'ensemble du site.

## Implémentations des hooks de thème

Le thème implémente plusieurs fonctions hook pour contrôler la sélection des templates et la préparation des données.

### Hooks de suggestions de templates

Le thème fournit des suggestions de templates personnalisées via des implémentations de `hook_theme_suggestions_HOOK_alter` :

| Fonction hook | Rôle | Logique clé |
|---|---|---|
| `ad_theme_theme_suggestions_taxonomy_term_alter` | Variantes de templates pour les termes de taxonomie | Ajoute des suggestions par mode d'affichage et ID de terme |
| `ad_theme_theme_suggestions_views_view_alter` | Variantes de templates pour les vues | Ajoute des suggestions par ID de vue et affichage |
| `ad_theme_theme_suggestions_page_title_alter` | Variantes de templates pour les titres de page | Suggestions spécifiques aux routes (recherche, newsletter, etc.) |
| `ad_theme_theme_suggestions_views_view_unformatted_alter` | Templates de vues non formatées | Suggestions spécifiques à l'affichage |
| `ad_theme_theme_suggestions_form_element_alter` | Templates d'éléments de formulaire | Suggestions pour les éléments du formulaire de recherche |
| `ad_theme_theme_suggestions_input_alter` | Templates d'éléments input | Suggestions pour les champs et boutons de soumission de recherche |
| `ad_theme_theme_suggestions_fieldset_alter` | Templates de fieldsets | Suggestions pour les fieldsets rich select |

### Fonctions de preprocessing

#### Preprocessing du nœud film complet

La fonction `ad_theme_preprocess_node__movie__full` effectue :

1. **Chargement de la configuration** : charge la configuration depuis les config pages pour les titres de blocs et libellés
2. **Correspondance des collections** : trouve les collections qui incluent ce film et doivent s'afficher sur les pages films
3. **Tri des offres** : trie les offres par valeur `field_taxo_order`
4. **Tri des partenaires** : dans chaque offre, trie les partenaires par leur ordre
5. **Filtrage des fenêtres de droits** : supprime les partenaires dont les droits ont expiré ou sont futurs
6. **Collecte des cache tags** : agrège les cache tags des termes partenaires pour une invalidation correcte

#### Preprocessing de la carte film

La fonction `ad_theme_preprocess_node__movie__card` prépare les informations de disponibilité partenaire pour l'affichage en carte :

- Trie les offres et partenaires par ordre
- Extrait le premier partenaire disponible
- Formate la date de fin pour l'affichage
- Fournit l'attribut de prononciation pour les lecteurs d'écran

#### Preprocessing des paragraphes

Plusieurs fonctions de preprocessing gèrent les entités paragraphe :

* `ad_theme_preprocess_paragraph__pg_paragraph__full`
* `ad_theme_preprocess_paragraph__pg_movies__full`
* `ad_theme_preprocess_paragraph__pg_card__full`
* `ad_theme_preprocess_paragraph__pg_horizontal_card__full`
* `ad_theme_preprocess_paragraph__pg_video_iframe__full`

Toutes les fonctions de preprocessing de paragraphes extraient les données d'un CTA (call-to-action) :

- URL depuis `field_pg_link`
- Texte du lien (title)
- Flag lien externe depuis `field_pg_is_external`
- Attribut target (`_blank` pour externe, `_self` pour interne)
- Style du bouton depuis `field_pg_style`

#### Preprocessing des éléments de formulaire

Les fonctions `ad_theme_preprocess_form_element__select` et `ad_theme_preprocess_checkboxes` préparent :

- Titres au singulier et au pluriel pour la grammaire française
- Information de genre (`#is_female`)
- Options sous forme de chaîne JSON
- Valeurs sélectionnées sous forme de chaîne JSON
- Type de plugin pour les composants personnalisés

Ces éléments de formulaires sont des web components Lit.js. 

## Architecture SCSS

Le thème utilise une structure SCSS modulaire compilée vers `dist/styles.css`.

### Organisation des fichiers

* `/base/` : pour les styles de base (souvent pour les éléments `html`ou `body`)
* `/layout` : pour la présentation
* `/libs/` : pour importer des librairies externes
* `/modules/` : pour toutes les pages et tous les composants du projet
* `/states/`: pour les états
* `/themes/` : pour les différents thèmes (dark theme / light theme par exemple)
* `/tools/` : pour les outils dont les variables, les fix du Système de Design de l'État


### Propriétés CSS personnalisées

Le thème utilise les propriétés CSS du DSFR pour un thème cohérent :

| Propriété | Usage |
|---|---|
| `--background-default-grey` | Fonds de blocs par défaut |
| `--background-alt-blue-france` | Fonds de blocs alternatifs |
| `--background-contrast-grey` | Fonds des champs de formulaire |
| `--text-default-grey` | Couleur de texte par défaut |
| `--text-active-blue-france` | Couleurs des liens |
| `--border-plain-grey` | Couleurs des bordures |
| `--text-mention-grey` | Texte secondaire (dates de disponibilité) |

### Nommage des classes

Les noms des classes sont construites à partir des conventions de BEM (Blocks – Elements – Modifiers).

__Attention, toutes les classes sont préfixées de "audiodesc-".__

__Au début du projet, le préfixe était "ad-" mais certains blocs de contenus étaient masqués par les bloqueurs de pub ("ad" signifie "publicité" en anglais).__

## Composants JavaScript

### Composant PartnersSelect
Ce composant permet d'afficher le filtre "Partenaires" sur les pages de recherche.

La classe `PartnersSelect` fournit :

- **Pattern Manager** : le singleton `PartnersSelectManager` gère toutes les instances
- **Enregistrement** : les composants s'enregistrent au `connectedCallback`
- **Désenregistrement** : les composants se désenregistrent au `disconnectedCallback`
- **Gestion de la fermeture** : le manager peut fermer tous les dropdowns sauf celui actif
- **Shadow DOM désactivé** : utilise `createRenderRoot()` pour restituer dans le DOM léger

### Mise à jour live AJAX de la recherche

Le behavior Drupal gère l'accessibilité pour les résultats de recherche : il copie le texte des résultats vers une live region pour les annonces des lecteurs d'écran.

## Intégration de la configuration

Le thème charge la configuration depuis les entités Config Pages pour fournir des textes et paramètres éditables.

### Chargement conditionnel de Matomo

La fonction `ad_theme_preprocess_html` vérifie le champ `field_enable_matomo` depuis la config page patrimony et définit une variable `matomo` si activé. 
Cela permet aux templates d'inclure conditionnellement les scripts de tracking Matomo.

## Design responsive

Le thème implémente un design mobile-first avec les breakpoints DSFR.

### Système de breakpoints

| Breakpoint | Largeur min | Usage |
|---|---|---|
| Small (sm) | 576px | Boutons de formulaire, ajustements de grille |
| Medium (md) | 768px | Colonnes de grille, mises en page de cartes, logo opérateur du header |
| Large (lg) | 992px | Grille cartes partenaires (3 colonnes), taille des cartes films |
| Extra Large (xl) | 1200px | Ajustements de largeur du contenu |

## Gestion du cache

### Cache tags dans le preprocessing

Dans `ad_theme_preprocess_node__movie__full` les cache tags des termes de taxonomie partenaires sont agrégés et attachés à la page. 
Cela garantit que lorsqu'un terme de taxonomie partenaire est mis à jour (logo, condition, ordre), toutes les pages films affichant ce partenaire sont invalidées.

## Implémentation de l'accessibilité

Le thème implémente les fonctionnalités d'accessibilité via les composants DSFR et des améliorations personnalisées.

### Liens d'évitement

Des liens d'évitement sont disponibles pour la navigation au clavier vers :

- Contenu (`#contenu`)
- Recherche (`#search`)
- Navigation (`#navigation-main`)
- Pied de page (`#footer`)

### HTML sémantique

Le thème utilise des éléments sémantiques appropriés :

- `<header role="banner">`
- `<main role="main">`
- `<footer role="contentinfo">`
- `<nav>` pour les menus de navigation
- hiérarchie de titres correcte (`h1` / `h2` / `h3`)
- `<button>` ou `<a>`
