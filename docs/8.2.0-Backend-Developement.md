# Développement Backend

Cette page constitue un guide pratique pour le développement PHP backend du Portail de l'audiodescription. 
Elle couvre à la fois le **backend Symfony (Patrimony)** et le **backend Drupal**, notamment l'architecture des services, la gestion des entités, les patterns d'importeurs et les commandes personnalisées.

Pour l'ajout de nouvelles intégrations partenaires, voir [Intégrer un nouveau partenaire](8.3.0-Adding-New-Partners-Integrations.md). 
Pour le développement frontend (SCSS, JavaScript, Twig), voir [Développement Frontend](8.1.0-Frontend-Development.md).

## Vue d'ensemble de l'architecture

Le système repose sur une **architecture dual backend** avec des responsabilités distinctes :

- **Patrimony (Symfony)** : gère l'import de données depuis les partenaires externes, stocke les données des films, expose des endpoints API.
- **Drupal** : gère le contenu public, fournit les fonctionnalités CMS, synchronise les données depuis Patrimony.

## Développement backend Symfony (Patrimony)

### Gestion des films

Les entités Doctrine dans Patrimony suivent les conventions d'API Platform avec des groupes de sérialisation pour l'exposition API.

**Entité principale : Movie**

L'entité `Movie` (`patrimony/src/Entity/Patrimony/Movie.php`) est l'enregistrement d'un film, avec les identifiants externes de plusieurs partenaires.

**Configuration API Platform :** Les entités utilisent des attributs pour l'exposition API (lignes 16–30) :

- `#[GetCollection]` expose les endpoints de collection
- `#[Groups]` contrôle le contexte de sérialisation
- `#[ApiFilter]` active le filtrage (ex. `DateFilter`)

### Création des importeurs partenaires

Les importeurs partenaires implémentent l'interface `MovieImporterInterface` et suivent un pattern cohérent.

**Contrat de l'interface :**

```php
interface MovieImporterInterface
{
    public function import(bool $createMovies = false): void;
}
```

**Pattern d'implémentation** (cf. `FranceTvApiImporter.php`, lignes 27–310) :

1. **Injection par constructeur** : injection des services nécessaires via l'injection de dépendances
2. **Initialisation partenaire/offre** : création ou récupération des entités Partner et Offer
3. **Nettoyage des données** : suppression des solutions existantes pour le partenaire
4. **Requête API** : récupération des données depuis l'API externe
5. **Transformation des données** : mapping de la réponse API vers la structure interne
6. **Création d'entités** : création des entités `SourceMovie` et `Solution`
7. **Matching film** : liaison avec le `Movie` canonique via `MovieFetcher`


### Commandes console Symfony

Les commandes sont définies dans `src/Command/` et enregistrées via l'autoconfiguration.

**Structure d'une commande :**

```php
#[AsCommand(name: 'app:import:francetv')]
class ImportFranceTvCommand extends Command
{
    public function __construct(
        private readonly ImporterFactory $importerFactory,
    ) {
        parent::__construct();
    }

    protected function execute(InputInterface $input, OutputInterface $output): int
    {
        $this->importerFactory->create(ImportSourceType::FRANCE_TV)->import();
        return Command::SUCCESS;
    }
}
```

**Exécution via le Makefile** :

```bash
make pt-import-francetv
# équivaut à : docker compose exec -T patrimony php bin/console app:import:francetv
```

### Configuration des services

Les services sont configurés dans `config/packages/patrimony.yaml`.

## Développement backend Drupal

Le code spécifique au Portail de l'Audiodescription est dans : 
- le module `modules/custom/audiodescription/`
- le thème `themes/ad_theme/`

### Définition des services

Les services sont définis dans `audiodescription.services.yml`.

**Exemple de définition de service :**

```yaml
audiodescription.movie_manager:
  class: Drupal\audiodescription\EntityManager\MovieManager
  arguments:
    - '@entity_type.manager'
    - '@audiodescription.partner_manager'
    - '@logger.channel.audiodescription'
```

### Importeurs de synchronisation Patrimony

Les importeurs récupèrent les données depuis l'API Patrimony et créent ou mettent à jour les entités Drupal.

### Contrôleurs personnalisés

Les contrôleurs gèrent les requêtes HTTP et retournent des tableaux de rendu ou des réponses.

**Définition des routes** (`audiodescription.routing.yml` :

```yaml
audiodescription.movie_search:
  path: '/films'
  defaults:
    _controller: '\Drupal\audiodescription\Controller\MovieSearchController::index'
    _title: 'Recherche de films'
  requirements:
    _permission: 'access content'
```

**Pattern d'un contrôleur :**

```php
class MovieSearchController extends ControllerBase
{
    public function __construct(
        private readonly SearchManager $searchManager,
    ) {}

    public function index(Request $request): array
    {
        return [
            '#theme' => 'movie_search',
            '#results' => $this->searchManager->search($request->query->all()),
        ];
    }
}
```

### Commandes Drush

Les commandes Drush fournissent un accès CLI aux fonctionnalités Drupal.

**Enregistrement des commandes** (via `drush.services.yml`) :

```yaml
audiodescription.import_all_patrimony_command:
  class: Drupal\audiodescription\Drush\Commands\ImportAllPatrimonyCommand
  arguments:
    - '@audiodescription.movie_patrimony_importer'
  tags:
    - { name: drush.command }
```

**Implémentation de la commande** (`ImportAllPatrimonyCommand.php`, lignes 1–96) :

```php
class ImportAllPatrimonyCommand extends DrushCommands
{
    #[CLI\Command(name: 'audiodescription:import-all', aliases: ['adia'])]
    public function importAll(): void
    {
        $this->moviePatrimonyImporter->import();
        $this->logger()->success('Import terminé.');
    }
}
```

## Gestion des erreurs

### Système d'erreur 410 Gone

Le système implémente une gestion personnalisée des erreurs 410 pour les films devenus indisponibles (supprimés du catalogue).

**Event Subscriber** (`MovieExceptionSubscriber.php` :

```php
class MovieExceptionSubscriber implements EventSubscriberInterface
{
    public static function getSubscribedEvents(): array
    {
        return [KernelEvents::EXCEPTION => 'onException'];
    }

    public function onException(ExceptionEvent $event): void
    {
        $exception = $event->getThrowable();
        if ($exception instanceof MovieGoneException) {
            $response = new Response(
                $this->twig->render('error-410.html.twig'),
                Response::HTTP_GONE,
            );
            $event->setResponse($response);
        }
    }
}
```

**Enregistrement du service** (`audiodescription.services.yml`) :

```yaml
audiodescription.movie_exception_subscriber:
  class: Drupal\audiodescription\EventSubscriber\MovieExceptionSubscriber
  tags:
    - { name: event_subscriber }
```
