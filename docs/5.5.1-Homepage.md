# Page d'accueil

## Objectif et périmètre

Ce document explique l'architecture de la page d'accueil, incluant le `HomepageController` qui agrège le contenu, l'entité Config Pages qui stocke la configuration et le contenu de la page d'accueil, et le template qui restitue la page finale. 
Ce système permet un contrôle éditorial sur l'ensemble du contenu de la page d'accueil sans nécessiter de modifications de code.

Pour les informations sur les blocs films affichés sur la homepage (films nouveaux/bientôt indisponibles), voir [Collections & pages de genres (3.5.2)](https://deepwiki.com/betagouv/audiodescription/3.5.2-collections-and-genre-pages). 
Pour l'intégration du formulaire de recherche, voir [Formulaire de recherche et filtres](5.4.1-Search-Fom-Filters.md).

## Architecture du système

La page d'accueil utilise une architecture à 3 niveaux : 
1. une entité Config Pages qui stocke le contenu éditable
2. un contrôleur qui agrège ce contenu avec des données dynamiques
3. un template Twig qui restitue la sortie finale.

## Implémentation du HomepageController

Le `HomepageController` sert de gestionnaire de route pour `/homepage`.

### Dépendances de services

Le contrôleur utilise l'injection de dépendances pour accéder à trois services core :

| Service | Interface | Usage |
|---|---|---|
| `entity_type.manager` | `EntityTypeManagerInterface` | Charge et restitue les entités bloc |
| `config_pages.loader` | `ConfigPagesLoaderServiceInterface` | Charge l'entité config page homepage |
| `form_builder` | `FormBuilderInterface` | Construit le formulaire de recherche |

### Pattern d'extraction des données

Le contrôleur suit un pattern cohérent pour extraire les données depuis les champs de la config page.

## Structure de l'entité Config Pages

Le contenu de la homepage est stocké dans une entité Config Pages de type `homepage`. Elle fournit une interface de configuration singleton accessible à `/admin/config/system/config-pages/homepage`.

### Organisation des champs

Les champs sont organisés en 5 groupes logiques via le module Field Group :

| Groupe | Champs | Rôle |
|---|---|---|
| **Header** (`group_hp_header`) | `field_header_title`, `field_header_chapo`, `field_header_with_search_bar`, `field_header_cta`, `field_header_image` | Section hero avec titre, description, recherche optionnelle et CTA |
| **Films gratuits** (`group_free_movies`) | `field_new_movies_title`, `field_near_end_movies_title` | Titres des sections de blocs films (contenu chargé via les blocs) |
| **Newsletter** (`group_hp_newsletter`) | `field_newsletter_title`, `field_newsletter_description`, `field_newsletter_cta`, `field_newsletter_icon` | Section d'inscription à la newsletter |
| **Informations** (`group_hp_infos`) | `field_infos_title`, `field_infos_description`, `field_infos_image`, `field_infos_ctas` | Section informations avec plusieurs CTAs |
| **À propos** (`group_hp_about`) | `field_about_title`, `field_about_description`, `field_about_icon`, `field_about_pre_contact`, `field_about_email`, `field_about_button_text`, `field_about_subbutton_text` | Section contact avec intégration email |

## Sections de la homepage

Le contrôleur assemble 5 sections principales, chacune extraite de la config page et transmise au template.

### Section Header

La section header fournit la zone hero avec une fonctionnalité de recherche optionnelle.

### Section Newsletter

La section newsletter promeut l'inscription par email avec un unique CTA (call to action).

La section inclut :

- `title` : titre de la section (`field_newsletter_title`)
- `description` : contenu texte avec HTML (`field_newsletter_description`)
- `cta` : bouton d'appel à l'action unique (depuis le paragraphe `field_newsletter_cta`)
- `icon` : image décorative (`field_newsletter_icon`)

### Section Infos

La section infos affiche des informations générales avec plusieurs CTAs. 
Le contrôleur itère sur tous les paragraphes CTA et construit un tableau de configurations de boutons. 
Le template utilise ensuite le pattern bouton DSFR pour restituer chaque CTA.

### Section À propos

La section à propos fournit les informations de contact avec un lien email.

Spécificités de cette section :

- `email` : adresse email stockée dans `field_about_email`
- `button_text` : texte du bouton mailto
- `subbutton_text` : texte d'aide affiché sous le bouton
- `pre_contact` : texte affiché avant le bouton email

Le template construit un lien `mailto:` en utilisant la valeur du champ email.

> **Attention** : l'adresse e-mail de contact est longue. Il faut vérifier qu'elle puisse d'afficher sur une seule ligne, en particulier sur la version mobile.

## Intégration des blocs films

La page d'accueil affiche 2 blocs films dynamiques chargés comme entités séparées.

### Chargement des blocs

Le contrôleur charge les entités bloc préconfigurées et les restitue via le view builder des blocs. 
Les blocs eux-mêmes contiennent des Views qui interrogent les nœuds films selon des critères de disponibilité spécifiques.

## Rendu du template

### Patterns du template

| Pattern | Usage |
|---|---|
| `include('ui_suite_dsfr:button')` | Rendu d'un bouton unique |
| `include('ui_suite_dsfr:button_group')` | Plusieurs boutons |
| `class2wysiwygP('fr-text--lead')` | Formatage du texte WYSIWYG |

### Mise en page responsive

Le template utilise les classes de grille DSFR pour les mises en page responsives.

## Gestion du cache

### Cache tags

3 types de cache tags sont appliqués :

1. Cache tag de la config page homepage (ex. `config_pages:homepage:1`)
2. Cache tags des entités bloc
3. Cache tags des nœuds films référencés

Les cache tags sont appliqués à la fois dans le tableau `#cache` et via `CacheableMetadata::addCacheTags()`.

### Cache tag de la config page

Le cache tag de la config page est récupéré dynamiquement et retourne un tag du type `config_pages:homepage:1`, garantissant que la homepage est reconstruite chaque fois qu'un administrateur modifie la configuration de la homepage.
