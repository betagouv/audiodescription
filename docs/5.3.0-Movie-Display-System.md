# Système d'affichage des films

Ce document explique comment les films sont affichés dans différents contextes du frontend Drupal : pages de film complètes, affichages en carte, teasers et résultats de recherche. 
Le système utilise plusieurs modes d'affichage avec des templates spécialisés et une logique de preprocessing pour afficher conditionnellement les informations des films selon la disponibilité de l'audiodescription, les offres partenaires et le contexte de rendu.

Pour les détails sur la structure des templates et leurs variantes, voir [Templates de nœuds film (3.3.1)](https://deepwiki.com/betagouv/audiodescription/3.3.1-movie-node-templates). 
Pour la logique de rendu des partenaires et offres, voir [Affichage des offres & partenaires (3.3.2)](https://deepwiki.com/betagouv/audiodescription/3.3.2-offer-and-partner-display). 
Pour l'affichage des films dans les résultats de recherche, voir [Recherche & Filtrage (3.4)](https://deepwiki.com/betagouv/audiodescription/3.4-search-and-filtering).

## Modes d'affichage et contextes

Le système d'affichage des films implémente 5 modes d'affichage principaux, chacun optimisé pour des contextes spécifiques :

| Mode d'affichage | Template | Contexte principal                                         |
|---|---|------------------------------------------------------------|
| `full` | `node--movie--full.html.twig` | Page de détails du film                                    |
| `card` | `node--movie--card.html.twig` | Collections de la page d'accueil, des pages d'informations |
| `teaser` | `node--movie--teaser.html.twig` | Pages de taxonomie de collections                          |
| `card-result` | `node--movie--card-result.html.twig` | Résultats de recherche (titre h3)                          |
| `card-result-h4` | `node--movie--card-result-h4.html.twig` | Résultats de recherche (titre h4)                          |

## Architecture des templates

Le système de thème Drupal associe les routes aux modes d'affichage, puis applique les suggestions de templates. 
Les fonctions de preprocessing s'exécutent avant le rendu des templates pour préparer les données structurées.

## Extraction des données et preprocessing

La couche de preprocessing transforme les champs bruts des nœuds en variables structurées pour les templates. 
2 fonctions de preprocessing clés gèrent la logique d'affichage la plus complexe.

### ad_theme_preprocess_node__movie__full

Cette fonction prépare les données pour la vue complète du film :

- Trie les offres par `field_taxo_order` (du plus petit au plus grand)
- Filtre les partenaires avec des plages de dates invalides par rapport à la date courante
- Regroupe les partenaires sous leurs types d'offres
- Collecte les cache tags pour une invalidation correcte

### ad_theme_preprocess_node__movie__card

Cette fonction prépare l'affichage du partenaire principal pour les vues carte.

**Objectif** : les vues carte n'affichent que le **partenaire le plus prioritaire** de l'**offre la plus prioritaire**. 
Cela est déterminé par le champ `field_taxo_order` sur les termes de taxonomie offre et partenaire.


## Mapping des champs et patterns d'affichage

Tous les templates films extraient les données depuis des champs de nœuds cohérents avec des patterns d'affichage communs.

### Champs principaux du film

| Nom du champ | Type | Logique d'affichage                                   |
|---|---|-------------------------------------------------------|
| `field_has_ad` | Booléen | /                                                     |
| `field_poster_external` | Chaîne (URI) | URL d'image externe, préférée à `field_poster`        |
| `field_title` / `title` | Chaîne | Affiché via `{{ label }}` ou `{{ node.title.value }}` |
| `field_synopsis` | Texte | Tronqué à 315 caractères dans les vues carte          |
| `field_production_year` | Entier | Formaté "sorti en **ANNÉE**"                          |
| `field_duration` | Entier (minutes) | Converti au format "X h YY"                           |
| `field_directors` | Référence d'entité | Multiples, affichés en liste séparée par des virgules |
| `field_genres` | Référence d'entité | Multiples, rendus comme tags                          |
| `field_public` | Référence d'entité | Unique, affichage du public                           |
| `field_offers` | Référence d'entité (paragraphe) | Structure imbriquée complexe                          |

### Pattern d'affichage des réalisateurs et genres

Tous les templates suivent le même pattern pour extraire les références d'entités multi-valeurs (itération sur les items du champ).

## Logique d'affichage conditionnelle

### Badge audiodescription

Tous les modes d'affichage affichent un badge indiquant la disponibilité de l'audiodescription. 
Le texte du badge varie selon la valeur de `field_has_ad`.

## Comparaison des modes d'affichage

### Vue complète (node--movie--full.html.twig)

La vue complète est la page de détails du film, affichant toutes les informations.

**Structure :**

1. **Section hero** : affiche (gauche), métadonnées (droite)
    - Année de production, durée, genres (comme tags cliquables), réalisateurs, public
    - Synopsis dans une section dédiée
2. **Bloc audiodescription** : badge `field_has_ad`, icônes de collections
3. **Bloc plateformes** : regroupé par type d'offre, affichant tous les partenaires valides

**Fonctionnalités exclusives :**
- Affiche **toutes les offres** regroupées par type (ex. "Gratuit", "SVOD")
- Affiche **tous les partenaires** pour chaque offre avec les plages de dates
- Affiche les icônes de collections pour les films dans des collections
- Inclut les champs durée et public

### Vue carte (node--movie--card.html.twig)

Utilisée sur la page d'accueil et les pages d'informations, affiche des informations condensées.

**Structure :**
- Titre
- Phrase avec les réalisateurs et l'année de production
- Jusqu'à 3 tags de genres
- Informations de disponibilité du partenaire principal (préprocessées)

**Affichage de la disponibilité** : la variable `partner` est définie par `ad_theme_preprocess_node__movie__card` pour n'afficher que le partenaire le plus prioritaire.

### Vue teaser (node--movie--teaser.html.twig)

Utilisée sur les pages de termes de taxonomie de collections.

**Structure :**
- Mise en page horizontale avec affiche à droite
- Titre
- Badge audiodescription
- Phrase avec les réalisateurs et l'année de production
- Synopsis (tronqué à 315 caractères)
- Tags de genres (non cliquables)

**Troncature du synopsis** : le synopsis est tronqué à 315 caractères pour maintenir la cohérence visuelle des listes.

### Vue card-result (node--movie--card-result.html.twig)

**Différence clé** : nécessite **à la fois** `has_ad` et `field_offers` pour être cliquable. 
Cela garantit que les utilisateurs ne peuvent accéder qu'aux films disposant de plateformes actives pour les regarder.

**Tags de genres** : les tags genres renvoient vers la recherche avec ce genre présélectionné.

## Structure offres et partenaires

La vue complète affiche une structure imbriquée complexe pour la disponibilité partenaire. 
La fonction de preprocessing garantit que seuls les partenaires avec des plages de dates valides sont inclus dans `block_platforms.offers`.

Structure de rendu dans le template :

```
block_platforms.offers (tableau trié par order)
└── offer (type d'offre : Gratuit, SVOD, TVOD…)
    └── partners (tableau de partenaires)
        └── partner (nom, lien, dates de droits, prononciation)
```

## Styles et structure CSS

L'organisation des styles d'affichage des films en modules SCSS :

| Fichier | Rôle | Classes clés |
|---|---|---|
| `src/scss/modules/movie/_card.scss` | Style de la vue carte | `.ad-movie-card`, `.ad-movie-card__platform` |
| `src/scss/modules/partner/_partner-card.scss` | Style des cartes partenaires | `.ad-partner-card`, `.ad-partner-card__title` |

**Grille des cartes partenaires** : les partenaires sont affichés dans une grille responsive — 1 colonne sur mobile, 3 colonnes sur desktop.

## Gestion du cache

La fonction de preprocessing de la vue complète collecte les cache tags de tous les termes de taxonomie partenaires référencés. 
Cela garantit que les pages films sont invalidées lorsque les termes de taxonomie partenaires sont mis à jour (changement de logo, mise à jour de la prononciation, etc.).
