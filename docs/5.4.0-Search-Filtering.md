# Recherche et filtrage

Cette page documente le système de recherche et de filtrage de films dans le frontend Drupal. 
Elle couvre l'implémentation de la page de recherche, les formulaires de filtres avec mises à jour AJAX, la construction des requêtes via Elasticsearch, et la pagination. 
Le système permet aux utilisateurs de rechercher des films par texte libre et de filtrer par genre, plateforme partenaire et d'afficher uniquement les films gratuits.

Pour les informations sur la configuration de Search API et Elasticsearch, voir [Configuration Search API (3.4.3)](https://deepwiki.com/betagouv/audiodescription/3.4.3-search-api-configuration). 
Pour les détails de construction des requêtes du search manager, voir [Manager de recherche & construction de requêtes (3.4.2)](https://deepwiki.com/betagouv/audiodescription/3.4.2-search-manager-and-query-building). 
Pour l'implémentation du formulaire de filtres, voir [Formulaire de recherche & filtres (3.4.1)](https://deepwiki.com/betagouv/audiodescription/3.4.1-search-form-and-filters).

## Architecture du système

Le système de recherche suit une architecture en couches où le contrôleur coordonne le formulaire, le search manager et le rendu du template.

## Composants principaux

### MovieSearchController

Le `MovieSearchController` orchestre le rendu de la page de recherche. 
Il construit les parameter bags depuis la requête, délègue les requêtes au search manager, construit les formulaires et assemble le render array.

**Méthodes clés :**

| Méthode | Rôle                                                                                   |
|---|----------------------------------------------------------------------------------------|
| `search(Request $request)` | Gestionnaire de route principal qui coordonne toute la logique de la page de recherche |
| `getCurrentFilters(array $ids)` | Charge les noms des termes de taxonomie sélectionnés depuis leurs IDs pour l'affichage |

**Constantes de configuration :**

| Constante | Valeur | Rôle |
|---|---|---|
| `PAGE_SIZE` | 20 | Nombre de résultats par page |
| `PAGINATION_SIZE` | 2 | Nombre de pages affichées avant/après la page courante |

**Processus du contrôleur :**

1. Crée un `MovieSearchParametersBag` depuis les paramètres de requête
2. Calcule l'offset selon la page courante : `($params->page - 1) * PAGE_SIZE`
3. Appelle `movieSearchManager->queryMovies()` pour exécuter la recherche
4. Appelle `movieSearchManager->countAdMovies()` pour le comptage de tous les films avec audiodescription
5. Restitue chaque entité avec le mode d'affichage `card_result`
6. Construit la pagination si `pagesCount > 1`
7. Charge les formulaires `SimpleMovieSearchForm` et `FiltersMovieSearchForm`
8. Retourne le render array avec le thème `movie_search`


### FiltersMovieSearchForm

Le formulaire de filtres fournit des cases à cocher pour les genres, partenaires et une option de filtrage gratuit uniquement. 
Tous les filtres utilisent des callbacks AJAX pour mettre à jour les résultats sans rechargement de page.

**Définition des champs de filtre :**

| Champ | Type | AJAX | Description |
|---|---|---|---|
| `reset` | bouton | Non | Réinitialise tous les filtres |
| `is_free` | case à cocher | Oui | Filtre les films en accès gratuit uniquement |
| `partner` | cases à cocher | Oui | Filtre par plateforme de streaming |
| `genre` | cases à cocher | Oui | Filtre par taxonomie genre |

**Configuration des champs partenaire et genre :**

- Tous 2 utilisent `#ad_plugin_type => 'rich-select'` pour l'enrichissement JavaScript
- Tous 2 utilisent `#is_rich_select => TRUE` pour activer le composant rich select
- Le champ partenaire a `#is_female => TRUE` pour la grammaire française des labels
- Le champ genre a `#is_female => FALSE`
- Labels configurés avec `#singular_title` et `#plural_title`

**Flux du callback AJAX :** la méthode `searchMovies()` gère les changements de filtres :

1. Récupère la saisie utilisateur depuis le form state
2. Met à jour les paramètres de requête avec les nouvelles valeurs de filtres
3. Crée un nouveau `MovieSearchParametersBag` depuis la requête modifiée
4. Calcule l'offset : `($params->page - 1) * PAGE_SIZE`
5. Interroge les films via `movieSearchManager->queryMovies()`
6. Compte les films audiodécrits avec `movieSearchManager->countAdMovies()`
7. Restitue les entités avec le view builder en mode `card_result`
8. Construit la structure de pagination
9. Retourne une `AjaxResponse` avec deux commandes :
    - `ReplaceCommand` pour mettre à jour le conteneur `#ajax` avec les nouveaux résultats
    - `SearchAjaxUrlUpdateCommand` pour mettre à jour l'URL du navigateur sans rechargement

### MovieSearchManager

Le search manager construit les requêtes Elasticsearch et gère la logique de pagination. 
Il traduit les paramètres de filtres de haut niveau en conditions Search API.

**Dépendances :** `EntityTypeManagerInterface` pour charger les termes de taxonomie et nœuds, et `Index::load('movies')` pour l'index Search API des films.

**Logique du filtre films gratuits :**

1. Charger le terme de taxonomie avec `field_taxo_code == "FREE_ACCESS"` depuis le vocabulaire `Taxonomy::OFFER`
2. Exécuter une requête SQL brute pour trouver les IDs de partenaires avec :
    - Des paragraphes offre référençant le terme d'offre FREE_ACCESS
    - Des paragraphes partenaire avec des plages de dates valides (début < maintenant ET fin > maintenant)
3. Jointure via la structure de paragraphes : `node -> field_offers -> pg_offer -> pg_partners -> pg_partner`
4. Filtrage par conditions de date sur `field_pg_start_rights` et `field_pg_end_rights`
5. Retourner les IDs de partenaires distincts
6. Créer un groupe de conditions OR ajoutant chaque ID partenaire à la condition `field_pg_partner`

**Construction de la pagination :** la méthode `buildPagination()`

| Élément | Condition | URL |
|---|---|---|
| `first` | page != 1 | page=1 avec filtres |
| `prev` | page != 1 | page=courante-1 avec filtres |
| `pages[]` | Toujours | Tableau des pages visibles avec titres et URLs |
| `next` | page != pagesCount | page=courante+1 avec filtres |
| `last` | page != pagesCount | page=pagesCount avec filtres |

La constante `PAGINATION_SIZE` contrôle le nombre de pages affichées avant/après la page courante (défaut : 2).

### MovieSearchParametersBag

Un Plain Old PHP Object (POPO) qui encapsule tous les paramètres de recherche avec typage et validation.

**Propriétés :**

| Propriété | Type | Défaut | Description |
|---|---|---|---|
| `search` | string | '' | Requête de recherche plein texte |
| `page` | int | 1 | Numéro de page courant |
| `withAd` | bool | false | Filtre pour les films audiodécrits (actuellement inutilisé) |
| `genre` | array | [] | IDs de termes de taxonomie genre |
| `partner` | array | [] | IDs de termes de taxonomie partenaire |
| `isFree` | bool | false | Filtre pour les films en accès gratuit |

**Méthode factory statique :** `createFromRequest(Request $request)` extrait les paramètres depuis la chaîne de requête.

**Méthodes utilitaires :**

| Méthode | Rôle                                                                        |
|---|-----------------------------------------------------------------------------|
| `filtersToArray()` | Convertit les filtres sélectionnés en tableau pour la construction des URLs |
| `isEmptyParametersBag()` | Vérifie si tous les filtres sont à leurs valeurs par défaut                 |

## Construction de la requête de recherche

La requête Search API est construite progressivement selon les filtres actifs.

**Logique des groupes de conditions :**

- La recherche textuelle utilise la méthode `keys()` pour la correspondance plein texte
- Les filtres genre et partenaire utilisent des groupes OR : correspondance avec n'importe quelle valeur sélectionnée
- Les différents types de filtres sont combinés avec AND : tous les types de filtres actifs doivent correspondre
- Le filtre gratuit contraint indirectement les partenaires via les offres FREE_ACCESS validées par dates

## Rendu du template

**Variables clés du template :**

| Variable | Type | Description |
|---|---|---|
| `form` | render array | SimpleMovieSearchForm |
| `filtersForm` | render array | FiltersMovieSearchForm |
| `filters.search` | chaîne | Requête de recherche active |
| `filters.genres` | tableau | Noms de genres sélectionnés |
| `filters.plateformes` | tableau | Noms de partenaires sélectionnés |
| `filters['films gratuit uniquement']` | chaîne | "oui" ou null |
| `movies.total` | int | Nombre total de résultats |
| `movies.items` | tableau | Entités cartes film restituées |
| `movies.pagination` | tableau | Structure de pagination |
| `movies.totalWithAd` | int | Nombre de résultats avec audiodescription |
| `blockContact` | render array | Bloc de contact |

**Conteneur AJAX :** le `<div id="ajax">` est remplacé lors des changements de filtres. 

Il contient :
1. Titre des résultats avec compteur
2. Affichage des filtres actifs sous forme de tags avec bouton de suppression
3. Grille des films
4. Composant de pagination (si `pagesCount > 1`)

## Mécanisme de mise à jour AJAX

Le système met à jour les résultats de recherche sans rechargement de page grâce au module d'Ajax de Drupal.

**Commande ReplaceCommand :** cible `#ajax` et fournit le nouveau contenu HTML.

**Commande de mise à jour d'URL :** la `SearchAjaxUrlUpdateCommand` met à jour l'URL du navigateur avec l'état courant des filtres via l'API History, permettant :

- Les liens directs vers des résultats filtrés
- La navigation arrière/avant du navigateur
- La mise en favoris des états de recherche

**Bibliothèques JavaScript attachées :** :

- `audiodescription/ajax_filter_update` gère les interactions AJAX du formulaire de filtres
- `audiodescription/ajax_live_update` met à jour la live region ARIA pour l'accessibilité

## Système de pagination

La pagination est gérée par `MovieSearchManager::buildPagination()` qui crée une structure compatible avec le pattern de pagination DSFR.

**Logique de visibilité des pages :** :

1. Calculer `indexBefore = pageCourante - i` (i de 1 à PAGINATION_SIZE)
2. Calculer `indexAfter = pageCourante + i` (i de 1 à PAGINATION_SIZE)
3. Inclure `indexBefore` si `> 0`
4. Inclure `indexAfter` si `< pagesCount`
5. Trier les pages précédentes par ordre croissant
6. Ajouter la page courante (sans URL)
7. Ajouter les pages suivantes

**Construction des URLs :** toutes les URLs préservent l'état des filtres via `buildUrl()` garantissant que les paramètres genre, partenaire, gratuit et recherche persistent lors de la navigation paginée.

## Gestion de l'état des filtres

Les valeurs extraites du `ParametersBag` alimentent les propriétés `#default_value` des champs du formulaire afin que les filtres sélectionnés apparaissent cochés au chargement de la page.

## Configuration de l'index de recherche

Le système de recherche repose sur l'index Search API `movies` connecté à Elasticsearch.

**Champs indexés utilisés dans les requêtes :**

| Machine name du champ | Champ d'entité | Rôle |
|---|---|---|
| `field_has_ad` | `field_has_ad` | Filtrer les films audiodécrits |
| `field_genres` | `field_genres` | Référence de taxonomie genre |
| `field_pg_partner` | Champ de paragraphe imbriqué | Disponibilité partenaire |
| Recherche plein texte | Plusieurs champs | `keys()` cherche dans titre, synopsis, etc. |

**Traitement des résultats :** les IDs d'entités sont extraits des éléments de résultats. 
Les IDs suivent le format `entity:node/NID:LANGCODE`, nécessitant un parsing pour extraire l'ID de nœud.
