# Services

Ce document explique la configuration du conteneur d'injection de dépendances pour le module personnalisé audiodescription, incluant tous les services enregistrés, leurs dépendances, et la façon dont ils s'intègrent aux services de Drupal core. L'architecture des services suit les bonnes pratiques Drupal 8+ en matière d'injection de dépendances et d'organisation des services.

Pour les informations sur les implémentations spécifiques des importers, voir [Importers de synchronisation Patrimony (3.1.2)](https://deepwiki.com/betagouv/audiodescription/3.1.2-patrimony-sync-importers). Pour les services de gestion des erreurs, voir [Gestion des erreurs (3.1.3)](https://deepwiki.com/betagouv/audiodescription/3.1.3-error-handling). Pour les builders de fil d'Ariane, voir [Breadcrumb Builders (3.6.1)](https://deepwiki.com/betagouv/audiodescription/3.6.1-breadcrumb-builders).

## Vue d'ensemble des définitions de services

Tous les services du module audiodescription sont définis dans `audiodescription.services.yml`. 
Le fichier utilise la configuration YAML du conteneur de services de Drupal avec le paramètre `_defaults.autoconfigure: true`, qui enregistre automatiquement les classes de services avec les tags appropriés en fonction des interfaces implémentées.

Les services sont organisés en plusieurs groupes fonctionnels :

- **Importers Patrimony** : services qui synchronisent les données de la base Patrimony vers les entités Drupal
- **Entity Managers** : services qui encapsulent les opérations CRUD sur les entités avec la logique métier
- **Services de recherche** : services pour la recherche et le filtrage de films
- **Breadcrumb Builders** : services qui génèrent des fils d'Ariane contextuels
- **Event Subscribers** : services qui répondent aux événements du kernel Drupal
- **Services utilitaires** : parsers, loggers et extensions Twig

## Services Entity Manager

Les entity managers sont des services spécialisés qui encapsulent les opérations sur les entités avec une logique métier propre au domaine. 
Ils fournissent une couche cohérente au-dessus de l'`EntityTypeManagerInterface` de Drupal pour chaque type de contenu géré par le module.

| ID de service | Classe | Rôle                                                                               |
|---|---|------------------------------------------------------------------------------------|
| `audiodescription.manager.movie` | `MovieManager` | Gère les nœuds films, incluant la gestion des images d'affiche                     |
| `audiodescription.manager.director` | `DirectorManager` | Gère les termes de taxonomie réalisateur                                           |
| `audiodescription.manager.genre` | `GenreManager` | Gère les termes de taxonomie genre                                                 |
| `audiodescription.manager.offer` | `OfferManager` | Gère les termes de taxonomie offre                                                 |
| `audiodescription.manager.partner` | `PartnerManager` | Gère les termes de taxonomie partenaire                                            |
| `audiodescription.manager.nationality` | `NationalityManager` | Gère les termes de taxonomie nationalité                                           |
| `audiodescription.manager.public` | `PublicManager` | Gère les termes de taxonomie liée au public (par exemple : -10 ans, -12 ans, etc.) |
| `audiodescription.manager.actor` | `ActorManager` | Gère les termes de taxonomie acteur                                                |

Tous les entity managers dépendent de `@entity_type.manager`.

### Dépendances spécifiques du MovieManager

Le `MovieManager` est unique parmi les entity managers car il nécessite des services HTTP client et file repository pour télécharger et gérer les affiches de films depuis des URLs externes. 
Il dépend de `@audiodescription.manager.offer`, `@http_client` et `@file.repository` en plus de `@entity_type.manager`.

## Services importers Patrimony

Les importers Patrimony sont responsables de la synchronisation des données depuis la base de données backend Patrimony vers les entités Drupal. 
Chaque importer correspond à un type d'entité spécifique et orchestre la création/mise à jour des nœuds ou termes de taxonomie Drupal.

Tous les importers Patrimony dépendent de `@config_pages.loader` pour accéder à la configuration de la connexion à l'API fournie par Patrimony.

Le `MoviePatrimonyImporter` possède le graphe de dépendances le plus complexe, nécessitant tous les entity managers pour gérer l'entité film complète avec ses relations.

## Services d'import CSV

Deux services d'import CSV gèrent l'import initial de données depuis des fichiers CSV :

| ID de service | Classe | Paramètre fichier | Rôle |
|---|---|---|---|
| `audiodescription.cnc_csv_importer` | `CncCsvImporter` | `%cnc_movies_file%` | Importe les films depuis les données de référence CNC |
| `audiodescription.public_csv_importer` | `PublicCsvImporter` | `%cnc_publics_file%` | Importe les définitions de classifications |

Les deux services utilisent le service `@audiodescription.parser.csv` pour le parsing CSV.
Les chemins de fichiers sont définis comme paramètres du conteneur en bas du fichier de services (`audiodescription.services.yml`).

Le service `ImporterFactory` fournit une factory centralisée pour créer les instances d'importers.

## Architecture du service de recherche

La fonctionnalité de recherche est fournie par un service manager dédié :

**Service** : `audiodescription.manager.movie_search`  
**Classe** : `MovieSearchManager`  
**Dépendances** : `@entity_type.manager`

Ce service encapsule toute la construction des requêtes de recherche de films et le traitement des résultats, abstrayant l'intégration Search API et Elasticsearch des contrôleurs et formulaires. 
Il est utilisé par le `MovieSearchController` et le `FiltersMovieSearchForm` pour exécuter les recherches.

## Services Breadcrumb Builder

7 services breadcrumb builder fournissent une navigation contextuelle. Chaque builder est enregistré avec le tag `breadcrumb_builder` et une valeur de priorité qui contrôle l'ordre d'évaluation.
Les builders liés aux taxonomies (`collection`, `director`, `genre`) ont une priorité plus élevée (1010) que les builders de nœuds et de vues (100), garantissant qu'ils sont évalués en premier.

Le `NodeBreadcrumbBuilder` et le `ViewBreadcrumbBuilder` nécessitent des dépendances supplémentaires pour la traversée des liens de menu.

## Architecture des Event Subscribers

2 event subscribers gèrent les conditions d'erreur en écoutant `KernelEvents::EXCEPTION` :

### MovieExceptionSubscriber

**Service** : `audiodescription.exception_subscriber`  
**Classe** : `MovieExceptionSubscriber`  
**Priorité** : 200  
**Rôle** : intercepte les `AccessDeniedHttpException` pour les nœuds films dépubliés et les convertit en réponses 410 Gone

Ce subscriber vérifie si une exception d'accès refusé s'est produite sur un nœud film dépublié et restitue une page d'erreur 410 personnalisée à la place. 
Il dépend des services `@current_route_match` et `@renderer`.

### Error410ResponseSubscriber

**Service** : `audiodescription.error_410_response_subscriber`  
**Classe** : `Error410ResponseSubscriber`  
**Priorité** : -50  
**Rôle** : restitue le template final de la page d'erreur 410

Ce subscriber gère les `HttpException` avec le code de statut 410, restituant le template `error_410`. 
La priorité négative garantit qu'il s'exécute après les autres gestionnaires d'exceptions.

## Services d'extension Twig

2 filtres Twig personnalisés étendent les fonctionnalités des templates :

| ID de service | Classe | Nom du filtre | Rôle |
|---|---|---|---|
| `audiodescription.wysiwyg_chapo_twig_filter` | `WysiwygChapoTwigFilter` | `wysiwyg_chapo` | Formate le contenu WYSIWYG pour l'affichage en chapeau/texte d'accroche |
| `audiodescription.remove_prefix_number_twig_filter` | `RemovePrefixNumberTwigFilter` | `remove_prefix_number` | Supprime les préfixes numériques des noms de termes de taxonomie |

Les deux services sont tagués avec `twig.extension` pour un enregistrement automatique.

## Service de logging

Un canal de log dédié est configuré pour la journalisation spécifique au module :

**Service** : `logger.channel.audiodescription`  
**Parent** : `logger.channel_base`  
**Nom du canal** : `audiodescription`

Ce service étend le canal de log de base de Drupal et peut être injecté dans tout service ayant besoin de journaliser des messages spécifiques au module audiodescription.

## Patterns d'utilisation des services

Les services sont injectés dans différents composants Drupal :

**Dans les contrôleurs** : les contrôleurs reçoivent les services via l'injection de constructeur par la méthode `create()`. Exemple : `GenresController` injecte `@entity_type.manager`.
**Dans les commandes Drush** : les commandes Drush reçoivent les services en les déclarant dans l'annotation `@command` et en implémentant l'injection de constructeur. Des services comme `MoviePatrimonyImporter` sont utilisés pour exécuter les opérations d'import.
**Dans les event subscribers** : les event subscribers reçoivent les services via l'injection de constructeur et sont automatiquement enregistrés via le tag `event_subscriber`.
**Dans les breadcrumb builders** : les breadcrumb builders reçoivent les services via l'injection de constructeur et sont enregistrés via le tag `breadcrumb_builder` avec une priorité.

## Paramètres de configuration

Le fichier de services définit 2 paramètres pour les chemins des fichiers CSV :

| Paramètre | Valeur | Utilisation                                                                       |
|---|---|-----------------------------------------------------------------------------------|
| `cnc_movies_file` | `modules/custom/audiodescription/files/full-movies.csv` | Chemin du fichier CSV des données films CNC                                       |
| `cnc_publics_file` | `modules/custom/audiodescription/files/publics.csv` | Chemin du fichier CSV de liste des publics (par exemple : -10 ans, -12 ans, etc.) |

Ces paramètres sont injectés dans les services importers via la syntaxe `%nom_parametre%`.

## Intégration aux services Drupal core

Les services du module s'intègrent à plusieurs services core de Drupal :

| Service core | ID de service | Utilisé par | Rôle |
|---|---|---|---|
| Entity Type Manager | `@entity_type.manager` | Tous les entity managers, importers, contrôleurs | Opérations CRUD sur les entités |
| Config Pages Loader | `@config_pages.loader` | Tous les importers Patrimony | Accès à la config base Patrimony |
| HTTP Client | `@http_client` | `MovieManager` | Téléchargement des affiches externes |
| File Repository | `@file.repository` | `MovieManager` | Sauvegarde des fichiers téléchargés |
| Route Match | `@current_route_match` | `MovieExceptionSubscriber` | Accès au contexte de la route courante |
| Renderer | `@renderer` | Event subscribers, contrôleurs | Rendu des render arrays |
| Menu Link Tree | `@menu.link_tree` | `NodeBreadcrumbBuilder`, `ViewBreadcrumbBuilder` | Construction des fils d'Ariane basés sur les menus |

Ce pattern d'intégration garantit que le module suit les bonnes pratiques Drupal et exploite les fonctionnalités core plutôt que de réimplémenter des opérations communes.
