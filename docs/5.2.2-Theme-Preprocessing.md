# Preprocessing du thème

Ce document explique la couche de preprocessing du thème personnalisé `ad_theme`. 
Le preprocessing du thème est le mécanisme par lequel le code PHP prépare et transforme les données avant qu'elles soient transmises aux templates Twig pour le rendu. 
Cela inclut l'implémentation des suggestions de hooks de thème (pour la sélection des variantes de templates) et des fonctions de preprocessing (pour la préparation des données et la logique métier).

Pour les informations sur l'intégration du DSFR et les styles SCSS, voir [Intégration DSFR](5.2.1-DSFR-Integration.md). 
Pour les composants JavaScript côté client, voir [Composants JavaScript (3.2.3)](https://deepwiki.com/betagouv/audiodescription/3.2.3-javascript-components). 
Pour la consommation des données préprocessées dans les templates, voir [Templates de nœuds film (3.3.1)](https://deepwiki.com/betagouv/audiodescription/3.3.1-movie-node-templates).

## Objectif et périmètre

Le preprocessing du thème remplit 3 fonctions principales :

1. La **logique de suggestions de templates** détermine dynamiquement quelle variante de template Twig utiliser en fonction du type de contenu, du mode d'affichage, de la route et d'autres facteurs contextuels.
2. La **transformation des données** convertit les données brutes des entités en variables prêtes pour les templates, notamment le formatage des dates, le tri des collections, le filtrage par fenêtres de disponibilité et l'agrégation des entités liées.
3. La **séparation de la logique métier** maintient les manipulations complexes de données hors des templates en effectuant les calculs, requêtes et logiques conditionnelles en PHP.

## Système de suggestions de thème

Les suggestions de thème permettent à Drupal de sélectionner des variantes de templates de plus en plus spécifiques selon le bundle d'entité, le mode d'affichage et le contexte. Le thème implémente des hooks de suggestion qui ajoutent des patterns de nommage personnalisés.

### Hooks de suggestion implémentés

| Hook | Rôle | Pattern de template                           |
|---|---|-----------------------------------------------|
| `taxonomy_term_alter` | Suggestions par mode d'affichage pour les termes de taxonomie | `taxonomy_term__[bundle]__[view_mode]` |
| `views_view_alter` | Suggestions spécifiques à l'affichage des vues | `views_view__[view_id]__[display_id]`         |
| `page_title_alter` | Suggestions de titre de page spécifiques à la route | `page_title__[route]`, `page_title__[bundle]` |
| `views_view_unformatted_alter` | Suggestions pour les vues non formatées | `views_view_unformatted__[view_id]`           |
| `form_element_alter` | Suggestions d'éléments de formulaire spécifiques à la recherche | `form_element__search`                        |
| `input_alter` | Suggestions pour les champs de saisie de recherche | `input__search`, `input__submit__search`      |
| `fieldset_alter` | Suggestions de fieldset pour les rich select | `fieldset__rich_select`                       |

## Fonctions de preprocessing

Les fonctions de preprocessing s'exécutent après la sélection du template et avant le rendu. 
Elles préparent le tableau `$variables` qui sera disponible dans les templates Twig. 
Le thème implémente des fonctions de preprocessing pour les nœuds, les paragraphes, les termes de taxonomie, les vues et les éléments de formulaire.

## Gestion des cache tags

L'attribution correcte des cache tags garantit que les mises à jour de contenu invalident le cache de rendu. 

Par exemple, le preprocessing du film complet illustre ce pattern : les cache tags de tous les termes partenaires référencés sont agrégés et attachés à la page, garantissant que le cache de la page film est invalidé lorsqu'un terme partenaire est mis à jour (logo, condition, ordre d'affichage).

## Intégration avec les templates

Les variables préprocessées sont consommées dans les templates Twig via la syntaxe de variable standard. 

Par exemple, le template du film complet accède notamment à :

- `block_platforms.title` — valeur de config préprocessée
- `block_platforms.offers` — structure offre/partenaire triée et préprocessée
- `block_platforms.no_result` — message de repli préprocessé
