# Intégrer un nouveau partenaire

Ce document fournit un guide pas à pas pour intégrer un nouveau partenaire de contenu dans le Portail de l'audiodescription. 
Les intégrations partenaires récupèrent les données de films depuis des APIs externes ou des fichiers CSV et les importent dans le backend Patrimony.

Pour comprendre le fonctionnement du pipeline d'import existant, voir [Pipeline d'import des données](6.1.0-Pipeline-Import.md). 
Pour les détails des implémentations partenaires existantes, voir [Intégrations des APIs partenaires](6.1.1-Integration-API-partners.md). 
Pour la synchronisation des données de Patrimony vers Drupal, voir [Importers (synchronisation avec Patrimony)](5.1.2-Importers.md).

## Vue d'ensemble

L'ajout d'une nouvelle intégration partenaire nécessite des modifications à plusieurs niveaux du système :

1. Créer la classe importeur (`patrimony/src/Importer/Movie/`)
2. Ajouter le partenaire à l'enum `ImportSourceType` (`patrimony/src/Enum/ImportSourceType.php`)
3. Enregistrer l'importeur dans l'`ImporterFactory` (`patrimony/src/Importer/Movie/ImporterFactory.php`)
4. Ajouter les paramètres de configuration (`patrimony/config/packages/patrimony.yaml`)
5. Ajouter les enums Partenaire (`patrimony/src/Enum/PartnerCode.php`) et Offre (`patrimony/src/Enum/OfferCode.php`) si nécessaire
6. Ajouter le champ d'identifiant externe à l'entité `Movie` (`patrimony/src/Entity/Patrimony/Movie.php`)
7. Mettre à jour `MovieRepository` pour le matching par identifiant (`patrimony/src/Repository/MovieRepository.php`)
8. Créer la commande console (`patrimony/src/Command`)
9. Ajouter les cibles Makefile (`Makefile`)
10. Documenter l'intégration partenaire

## Étapes d'implémentation

### Étape 1 : Créer la classe importeur

Créer une nouvelle classe importeur dans `patrimony/src/Importer/Movie/` qui implémente l'interface `MovieImporterInterface`.


**Pattern d'implémentation clé :**

**1. Initialiser le Partenaire et l'Offre**

- Trouver ou créer l'entité `Partner` avec le `PartnerCode` approprié
- Trouver ou créer l'entité `Offer` avec l'`OfferCode` approprié

**2. Nettoyer les données existantes**

```php
// Marquer les films comme mis à jour
$this->movieRepository->updatedAllMovieWithPartner($partner);
// Supprimer les anciennes solutions
$this->solutionRepository->deleteAllByPartner($partner);
// Supprimer les anciens films sources
$this->sourceMovieRepository->deleteAllByPartner($partner);
```

**3. Récupérer et transformer les données**

- Effectuer la requête HTTP vers l'API (ou le fichier) partenaire
- Analyser la réponse (JSON ou CSV)
- Itérer sur chaque programme

**4. Créer le SourceMovie**

```php
// Utiliser SourceMovieManager::findOrCreate() avec titre, ID partenaire, année de production
$sourceMovie = $this->sourceMovieManager->findOrCreate(
    $title,
    $partner,
    $internalPartnerId,
    $productionYear,
);
// Définir les métadonnées : réalisateurs, casting, synopsis, durée, affiche, genres, classification
```

**5. Créer la Solution**

```php
// SolutionManager::createOrUpdate() avec partenaire, offre, lien, dates
$solution = $this->solutionManager->createOrUpdate(
    $partner,
    $offer,
    $link,
    $dateStartRights,
    $dateEndRights,
);
// Représente la disponibilité limitée dans le temps sur la plateforme partenaire
```

**6. Matcher avec le film canonique**

```php
// Construire le tableau des identifiants externes
$ids = [
    'franceTvId'    => $franceTvId,
    'imdbId'        => $imdbId,
    'plurimediaId'  => $plurimediaId,
    'allocineId'    => $allocineId,
];

// Utiliser MovieFetcher::fetchByIds() pour trouver le film canonique existant
$movie = $this->movieFetcher->fetchByIds($ids, $sourceMovie);
// Si non trouvé et option --create-movies activée, créer un nouveau Movie canonique
// Lier SourceMovie et Solution au Movie canonique
```

**7. Flush et clear**

```php
$this->entityManager->flush();  // Persister les modifications
$this->entityManager->clear();  // Libérer la mémoire
```


### Étape 2 : Ajouter le partenaire à l'enum `ImportSourceType`

Ajouter un nouveau cas à l'enum `ImportSourceType` pour représenter le nouveau partenaire.

**Fichier :** `patrimony/src/Enum/ImportSourceType.php`

```php
case NEW_PARTNER_API = 'NEW_PARTNER_API';
```

**Cas existants :**

```php
enum PartnerCode: string
{
    case CNC_CSV = 'CNC_CSV';
    case ARTE_TV_API = 'ARTE_TV_API';
    case CANAL_REPLAY_API = 'CANAL_REPLAY_API';

    case FRANCE_TV_API = 'FRANCE_TV_API';
    case LACINETEK_API = 'LACINETEK_API';
    case ORANGE_VOD_CSV = 'ORANGE_VOD_CSV';
    case TF1_API = 'TF1_API';

```

### Étape 3 : Enregistrer l'importeur dans l'`ImporterFactory`

Ajouter le nouvel importeur à l'instruction switch de l'`ImporterFactory`.

**Modifications requises :**

**1. Injection dans le constructeur**

```php
public function __construct(
    // ... importeurs existants ...
    private readonly NewPartnerApiImporter $newPartnerApiImporter,
) {}
```

**2. Instruction switch**

```php
return match($type) {
    // ... cas existants ...
    ImportSourceType::NEW_PARTNER_API => $this->newPartnerApiImporter,
};
```

### Étape 4 : Ajouter les paramètres de configuration

Ajouter les paramètres de configuration nécessaires pour l'authentification et les URLs de l'API.

**Fichier :** `patrimony/config/packages/patrimony.yaml`

```yaml
patrimony:
  # ... partenaires existants ...
  new_partner:
    api_url: '%env(NEW_PARTNER_API_URL)%'
    api_auth_token: '%env(NEW_PARTNER_API_AUTH_TOKEN)%'
```

**Variables d'environnement :** créer un `.env.local` avec les identifiants réels :

```dotenv
NEW_PARTNER_API_URL=https://api.newpartner.com/movies
NEW_PARTNER_API_AUTH_TOKEN=votre_token_ici
```

### Étape 5 : Ajouter les enums Partenaire et Offre

Si le partenaire nécessite de nouveaux types de partenaire ou d'offre, les ajouter aux enums correspondants.

**Enum PartnerCode :** `patrimony/src/Enum/PartnerCode.php`

```php
case NEW_PARTNER = 'NEW_PARTNER';
```

**Enum OfferCode :** `patrimony/src/Enum/OfferCode.php`

```php
// Ajouter uniquement si un nouveau type d'offre est nécessaire.
// La plupart des partenaires utilisent les types existants :
// FREE_ACCESS, SVOD, TVOD
case NEW_OFFER_TYPE = 'NEW_OFFER_TYPE';
```

### Étape 6 : Ajouter le champ d'identifiant externe à l'entité Movie

Si le partenaire fournit un identifiant externe unique, l'ajouter à l'entité `Movie` pour permettre le matching.

**Fichier :** `patrimony/src/Entity/Patrimony/Movie.php`

**Ajouter la propriété :**

```php
#[ORM\Column(nullable: true)]
#[Groups(['movie:read'])]
private ?string $newPartnerId = null;
```

**Ajouter le getter/setter :**

```php
public function getNewPartnerId(): ?string
{
    return $this->newPartnerId;
}

public function setNewPartnerId(?string $newPartnerId): self
{
    $this->newPartnerId = $newPartnerId;
    return $this;
}
```

**Créer la migration de base de données :**

```bash
make pt-migrate-diff   # Génère la migration
make pt-migrate        # Applique la migration
```

### Étape 7 : Mettre à jour MovieFetcher pour le matching par identifiant

Mettre à jour le `MovieRepository` pour prendre en compte le nouvel identifiant partenaire lors du matching des films.

**Fichier :** `patrimony/src/Repository/MovieRepository.php`

Ajouter l'identifiant partenaire à la logique de matching afin que les films puissent être dédupliqués entre partenaires via des identifiants externes communs (IMDB, Plurimedia, ou identifiants partenaire spécifiques).

```php
// Exemple : ajouter la recherche par newPartnerId
if (isset($ids['newPartnerId']) && !empty($ids['newPartnerId'])) {
            $qb = $qb->orWhere('m.newPartnerId = :newPartnerId')
                ->setParameter('newPartnerId', $ids['newPartnerId']);
        }
```

### Étape 8 : Créer la commande console

Créer une commande console pour déclencher l'importeur.

**Fichier :** `patrimony/src/Command/Import/NewPartnerApiImportCommand.php`

```php
#[AsCommand(
    name: 'app:import:new-partner',
    description: 'Import des films depuis l\'API Nouveau Partenaire',
)]
class NewPartnerApiImportCommand extends Command
{
    public function __construct(
        private readonly ImporterFactory $importerFactory,
    ) {
        parent::__construct();
    }

    protected function configure(): void
    {
        $this->addOption('create-movies', null, InputOption::VALUE_OPTIONAL, 'Créer les films canoniques manquants', false);
    }

    protected function execute(InputInterface $input, OutputInterface $output): int
    {
        $createMovies = $input->getOption('create-movies') === 'true';
        $this->importerFactory
            ->create(ImportSourceType::NEW_PARTNER_API)
            ->import(['create-movies' => $createMovies]);

        return Command::SUCCESS;
    }
}
```

### Étape 9 : Ajouter les cibles Makefile

Ajouter des cibles pratiques au Makefile pour l'exécution manuelle et automatisée.

**Fichier :** `Makefile`

**Cible d'import manuel :**

```makefile
pt-import-newpartner:
	docker compose exec -T patrimony php bin/console app:import:new-partner --create-movies=true
```

**Ajouter à l'import global :**

```makefile
pt-import-all: pt-import-francetv pt-import-artetv pt-import-tf1 pt-import-canal pt-import-lacinetek pt-import-newpartner
```

**Ajouter à l'import cron :**

```makefile
cron-import:
	# ... commandes existantes ...
	docker compose exec -T patrimony php bin/console app:import:new-partner --create-movies=true || true
```

pt-cron-import:
# ... commandes existantes ...
docker compose exec -T patrimony php bin/console app:import:new-partner --create-movies=true || true
```

> **Note :** Le flag `-T` désactive l'allocation de pseudo-TTY pour l'exécution cron. Le `|| true` garantit que le cron continue même si un import échoue.

## Guide de mapping des données

### Patterns de transformation des champs

| Champ            | Pattern | Exemple |
|------------------|---|---|
| **Réalisateurs** | Tableau d'objets avec `fullname` | `[{'fullname': 'Jane Doe'}, ...]` |
| **Casting**      | Tableau d'objets avec `fullname` et `role` optionnel | `[{'fullname': 'John Smith', 'role': 'Lead'}]` |
| **Genres**       | Tableau de chaînes | `['Drame', 'Comédie']` |
| **Synopsis**     | Supprimer le HTML, décoder les entités | `html_entity_decode(strip_tags($raw))` |
| **Durée**        | Convertir en minutes (entier) | ISO 8601 → `($h * 60) + $m` |
| **Public**       | Mapper vers 10/12/16/18/TP | Codes partenaire → codes standards |
| **Dates**        | Parser en DateTime | ISO 8601 → `new DateTime($string)` |
| **Affiche**      | Extraire l'URL pour le ratio approprié | Trouver l'image au format 3:4 |
