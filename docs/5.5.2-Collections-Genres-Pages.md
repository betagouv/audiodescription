# Pages de collections et de genres

Ce document couvre les systèmes d'affichage pour les termes de taxonomie Genre et Collection, incluant la page dédiée aux "Films par genres" (`/genres`) et les composants liés aux collections sur la page d'accueil. 
Les collections et les genres sont tous 2 des vocabulaires de taxonomie qui catégorisent les films : les genres représentent les catégories cinématographiques standard et les collections des regroupements éditoriaux.

Pour les informations sur l'architecture du contrôleur de la homepage et le système de config page, voir [Page d'accueil](5.5.1-Homepage.md). 
Pour les détails sur la synchronisation des termes de taxonomie depuis le backend Patrimony, voir [Importers (synchronisation avec Patrimony)](5.1.2-Importers.md). 
Pour le rendu des films dans ces pages, voir [Système d'affichage des films](5.3.0-Movie-Display-System.md).

## Vue d'ensemble du système

Le système des pages Collections et Genres comprend 3 composants principaux :

1. **Page "Films par genres"** (`/genres`) : une page dédiée affichant tous les termes genres comme tuiles
2. **Blocs collections de la homepage** : 2 types de blocs affichant les collections mises en avant et standard
3. **Pages de termes de collection** : des pages individuelles de termes de taxonomie pour les genres et collections

Genres et Collections sont tous deux des vocabulaires de taxonomie qui référencent des nœuds films. 
Le système gère le rendu de ces termes dans différents contextes avec des stratégies de cache appropriées.

## Page "Films par genres"

### Route et contrôleur

La page "Films par genres" est accessible à `/genres` et est gérée par la classe `GenresController`.

### Implémentation du contrôleur

La méthode `GenresController::list()` implémente la page "Films par genres" :

| Étape | Implémentation |
|---|---|
| Interroger les genres | Utilise EntityQuery sur le vocabulaire 'genre', trié alphabétiquement |
| Charger les termes | Itère sur les IDs de termes en chargeant chaque entité Term |
| Restituer en tuile | Utilise ViewBuilder pour restituer chaque terme avec le mode d'affichage 'tile' |
| Appliquer le cache | Définit le cache tag 'taxonomy_term_list' via CacheableMetadata |

Le contrôleur retourne un render array utilisant le thème `genres_list` :

```php
$build = [
  '#theme' => 'genres_list',
  '#genres' => $genres,
  '#title' => 'Films par genre',
  '#cache' => [
    'tags' => ['taxonomy_term_list'],
  ],
];
```

### Structure du template

Le template de la page "Films par genres" est défini suit le pattern de surcharge des templates Drupal : `drupal/web/themes/ad_theme/templates/page/genres-list.html.twig` (dans le thème `ad_theme`).

Le template du thème utilise les classes de grille DSFR pour créer une grille responsive de tuiles de genres. 
La variable `{{ genres }}` contient un tableau de termes de taxonomie pré-restitués, et la classe `fr-grid-row--gutters` ajoute l'espacement entre les tuiles.

### Visibilité du titre de page

La liste des genres a une gestion spéciale du titre de page. 
Le bloc de titre de page standard est explicitement masqué sur cette route via la configuration de visibilité. 
Cela permet au contrôleur de restituer le titre dans son template personnalisé plutôt que d'utiliser le bloc de titre standard.

## Système de collections de la page d'accueil

### Composants d'affichage des collections

La page d'accueil affiche les collections via plusieurs blocs thématiques définis dans `audiodescription.module` :

| Hook de thème | Chemin du template | Variables | Rôle |
|---|---|---|---|
| `hp_new_free_movies_block` | `templates/hp-new-free-movies-block` | `title`, `movies` | Films gratuits récemment ajoutés |
| `hp_near_end_free_movies_block` | `templates/hp-near-end-free-movies-block` | `title`, `movies`, `cta` | Films gratuits bientôt indisponibles |

## Invalidation du cache des taxonomies de collections

### Hook de mise à jour des films

Le système implémente une invalidation intelligente du cache lorsque des films sont mis à jour. 
Le hook `audiodescription_node_update()` invalide les caches des termes de collection lorsque leurs films référencés changent.

**Détails d'implémentation** :

1. **Vérification du type d'entité** — vérifie que l'entité est un nœud 'movie'
2. **Interrogation des collections** — trouve tous les termes de collection qui référencent le film mis à jour via `field_taxo_movies`
3. **Construction des cache tags** — crée un tableau de tags `taxonomy_term:{tid}`
4. **Invalidation** — utilise le service `cache_tags.invalidator` pour invalider toutes les pages de collections

### Référence du champ collection

Les collections référencent les films via le champ `field_taxo_movies` sur les termes de taxonomie du vocabulaire 'collection'. 
Ce champ de référence d'entité multi-valeurs permet aux contributeurs de sélectionner manuellement les films qui apparaissent dans chaque collection.

Lorsqu'un film d'une collection est mis à jour (titre modifié, disponibilité mise à jour, etc.), le cache du terme de collection est invalidé, garantissant que les pages de collections affichent toujours les informations à jour.
