# Newsletter

Le système de newsletter génère du contenu HTML pour un email hebdomadaire mettant en avant les films audiodécrits gratuits disponibles sur le Portail de l'audiodescription. 
Le système interroge la base de données Patrimony pour sélectionner des films selon trois catégories (nouveautés, bientôt indisponibles, sélection aléatoire) et les restitue dans un template HTML prêt pour l'envoi par email. 

Pour les informations sur le modèle de données films sous-jacent, voir [Modèle de données](6.2.0-Data-Model.md). 
Pour les détails sur l'import des données films depuis les partenaires, voir [Pipeline d'import des données](6.1.0-Pipeline-Import.md).

## Stratégie de sélection des films

La newsletter sélectionne exactement **16 films** répartis en trois catégories, avec des quotas par plateforme pour garantir une représentation diversifiée des contenus :

| Catégorie | Nombre | Critères de sélection | Objectif |
|---|---|---|---|
| Nouveaux films | 6 au total | `startRights` dans les 7 derniers jours, `endRights` au moins 7 jours devant | Mettre en avant les contenus récents |
| └─ Arte | max 2 | Nouveaux films spécifiques à la plateforme | Équilibrer la présence Arte |
| └─ TF1 | max 1 | Nouveaux films spécifiques à la plateforme | Inclure les contenus TF1 |
| └─ France TV | reste | Complète le quota à 6 au total | Source principale de contenu gratuit |
| Films bientôt indisponibles | 7 | `endRights` entre +3 et +15 jours, triés par expiration imminente | Créer l'urgence de visionnage |
| Films aléatoires | 4 | Actuellement disponibles, non sélectionnés ci-dessus | Découverte du catalogue |

Les constantes sont définies dans le service NewsletterContentGenerator.php :

```php
  TOTAL_MOVIES = 16;
  MAX_RANDOM_MOVIES = 4;
  MAX_NEW_ARTE_MOVIES = 2;
  MAX_NEW_TF1_MOVIES = 1;
  MAX_NEW_MOVIES = 6;
  MAX_NEAR_END_MOVIES = 7;
```

## Méthodes de requête du repository

### `findNewFreeMovies($platformCode = null)`

Récupère les films devenus disponibles dans les 7 derniers jours et restant disponibles au moins 7 jours supplémentaires.

**Logique de requête** :

- Jointures : `Movie` → `Solution` → `Offer` + `Partner`
- Filtres :
    - `offer.code = 'FREE_ACCESS'`
    - `solution.endRights >= maintenant + 7 jours`
    - `solution.startRights <= maintenant`
    - `solution.startRights >= maintenant - 7 jours` (nouveauté de la semaine)
    - `solution.link IS NOT NULL`
    - `movie.hasAd = true`
    - Optionnel : `partner.code = :platform` si `$platformCode` fourni
- Retourne : tableau d'entités `Movie` avec solutions, offres et partenaires chargés en eager loading

### `findNearEndFreeMovies($alreadySelected, $maxResult)`

Récupère les films gratuits bientôt indisponibles, triés par date d'expiration pour créer l'urgence.

**Logique de requête** :

- Filtres :
    - `offer.code = 'FREE_ACCESS'`
    - `solution.endRights IS NOT NULL`
    - `solution.endRights <= maintenant + 15 jours`
    - `solution.endRights >= maintenant + 3 jours`
    - `solution.link IS NOT NULL`
    - `movie.hasAd = true`
    - `movie NOT IN :excluded` (exclut le tableau `$alreadySelected`)
- Tri : `solution.endRights ASC` (expiration la plus proche en premier)
- Limite : `$maxResult`

### `findNotSelectedFreeMovies($alreadySelected)`

Récupère tous les films gratuits actuellement disponibles non encore sélectionnés pour les autres sections.

**Logique de requête** :

- Filtres :
    - `offer.code = 'FREE_ACCESS'`
    - `solution.endRights >= maintenant`
    - `solution.startRights <= maintenant`
    - `solution.link IS NOT NULL`
    - `movie NOT IN :excluded`
- Aucun tri ni limite appliqués (mélange et limitation effectués dans le contrôleur)

### `countFreeMovies()`

Compte le nombre total de films gratuits distincts actuellement disponibles avec audiodescription.

**Logique de requête** :

- Retourne : un entier représentant le nombre de `movie.id` distincts
- Utilisé pour afficher la taille totale du catalogue dans le pied de la newsletter

## Flux de génération de la newsletter

### Algorithme de sélection détaillé

Le service NewsletterContentGenerator.php implémente un algorithme de sélection en quatre phases :

**Phase 1 : Nouveaux films **

```
1. Requête films Arte nouveaux → shuffle → prendre MAX_NEW_ARTE_MOVIES
2. Requête films TF1 nouveaux → shuffle → prendre MAX_NEW_TF1_MOVIES
3. Calculer le reste = MAX_NEW_MOVIES - nombre_Arte - nombre_TF1
4. Requête films France TV nouveaux → shuffle → prendre le reste
5. Fusionner tous les tableaux → shuffle final
```

**Phase 2 : Films bientôt indisponibles **

```
Requête films bientôt indisponibles en excluant les sélections Phase 1, limité à MAX_NEAR_END_MOVIES
(Pré-triés par date d'expiration dans la requête du repository)
```

**Phase 3 : Films aléatoires **

```
1. Requête tous les films gratuits en excluant Phase 1 + Phase 2
2. Shuffle → prendre MAX_RANDOM_MOVIES
```

**Phase 4 : Statistiques (L56)**

```
Compter tous les films gratuits pour l'affichage dans le pied de page
```

La randomisation avec `shuffle()` garantit de la variété même lorsque les mêmes requêtes s'exécutent chaque semaine, évitant les newsletters répétitives si les catalogues partenaires restent stables.

## Structure du template HTML

### Organisation du template

Le template de newsletter newsletter/generator.html.twig génère un document HTML email complet avec des styles intégrés (pas de CSS externe) pour une compatibilité maximale avec les clients email.

### Affichage de chaque film

Chaque film dans la newsletter restitue les informations suivantes :

| Élément | Logique Twig | Format d'affichage |
|---|---|---|
| **Titre** | `{{ movie.title }}` | Titre H3, 24px |
| **Réalisateur** | Jointure du tableau `movie.directors` | "de [Réalisateur(s)]" si présent |
| **Année** | `{{ movie.productionYear }}` | "sorti en [année]" si présente |
| **Genres** | Logique d'agrégation complexe des genres | Genres principaux joints par "&" |
| **Synopsis** | `{{ movie.synopsis }}` | Paragraphe texte complet |
| **Lien de visionnage** | `movie.solutions[0].link` | "Regarder sur [Partenaire]" |

**Logique d'affichage des genres** :

Le template implémente une logique de déduplication des genres pour afficher les genres principaux :

1. Pour chaque genre dans `movie.genres`
2. Vérifier si le genre possède une relation `mainGenre`
3. Si oui, utiliser `mainGenre.name`, sinon utiliser `genre.name`
4. Dédupliquer en n'ajoutant que les noms de genres uniques au tableau
5. Joindre avec le séparateur " & "

Cela garantit que les sous-genres (ex. "Film noir") s'affichent via leur catégorie principale (ex. "Thriller"), évitant de surcharger les lecteurs avec des taxonomies trop spécifiques.

## Accès et distribution

### Routes d'accès à la newsletter

Deux routes définies dans le contrôleur :

| Route | Chemin | Méthode | Rôle |
|---|---|---|---|
| `newsletter_newsletter` | `/newsletter` | `newsletter()` | Page simple avec lien vers le générateur |
| `newsletter_generator` | `/newsletter/generator` | `generator()` | Génère et affiche le contenu HTML |

### Processus de distribution automatique

Le système génère le HTML et envoie un email hebdomadaire automatiquement grâce à la cron task `make pt-send-newsletter`.

**Intégration Brevo :** le template contient des attributs HTML spécifiques à Brevo :

- Attribut `sib_link_id="0"` sur les images
- Hébergement d'images via CDN Brevo
- Structure du template optimisée pour l'éditeur email de Brevo

La commande Symfony `app:send-weekly-newsletter` utilise le service BrevoCampaignService.php pour créer une campagne et envoyer l'e-mail.

Les identifiants de connexion à Brevo sont stockés dans le fichier .env : 
* BREVO_API_KEY 
* BREVO_LIST_ID
