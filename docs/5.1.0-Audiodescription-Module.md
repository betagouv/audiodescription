# Module Audiodescription

## Objectif et périmètre

Le module Audiodescription est le module Drupal personnalisé principal qui fournit les fonctionnalités essentielles du portail d'audiodescription. Il définit les routes personnalisées, les contrôleurs, les services, les hooks de thème et les event subscribers qui alimentent le site public.

Cette page documente la structure du module, ses points d'intégration avec Drupal core et les systèmes fondamentaux qu'il fournit. 

Pour la documentation détaillée des sous-systèmes spécifiques, voir :

- Architecture des services et injection de dépendances : [](https://deepwiki.com/betagouv/audiodescription/3.1-audiodescription-module)
- Synchronisation des données depuis le backend Patrimony : [](https://deepwiki.com/betagouv/audiodescription/3.1-audiodescription-module)
- Gestion des erreurs pour les films indisponibles : [](https://deepwiki.com/betagouv/audiodescription/3.1-audiodescription-module)

## Vue d'ensemble de la structure du module

Le module Audiodescription est situé dans `drupal/web/modules/custom/audiodescription/` et comprend les composants clés suivants.


### Intégration au système de thème

Le module implémente `hook_theme()` pour enregistrer des templates personnalisés pour différents types de pages et composants. 
Ces définitions de thème permettent au module de fournir un rendu personnalisé pour les pages qui ne suivent pas les patterns d'affichage de nœuds Drupal standards.

#### Définitions des hooks de thème

Les hooks de thème sont définis dans le fichier `audiodescription.module` (`function audiodescription_theme($existing, $type, $theme, $path)`) et se répartissent en plusieurs catégories :

| Catégorie | Hook de thème | Template | Rôle                                                                        |
|---|---|---|-----------------------------------------------------------------------------|
| **Pages** | `homepage` | `homepage.html.twig` | Mise en page principale de l'accueil avec collections et films mis en avant |
| | `genres_list` | `genres-list.html.twig` | Page de liste des termes de taxonomie genre (page "Films par genre")        |
| | `movie_search` | `movie-search.html.twig` | Page de résultats de recherche avec filtres                                 |
| | `movie_search_ajax` | `movie-search-ajax.html.twig` | Fragment de résultats de recherche chargé en AJAX                           |
| **Blocs** | `hp_highlighted_collections_block` | `hp-highlighted-collections-block.html.twig` | Affichage des collections en page d'accueil                                 |
| | `hp_collections_block` | `hp-collections-block.html.twig` | Liste des collections avec filtre par genre                                 |
| | `hp_new_free_movies_block` | `hp-new-free-movies-block.html.twig` | Liste des films gratuits récemment ajoutés                                  |
| | `hp_near_end_free_movies_block` | `hp-near-end-free-movies-block.html.twig` | Liste des films gratuits bientôt indisponibles                              |
| | `search_contact_block` | `search-contact-block.html.twig` | Bloc d'informations de contact sur la page recherche                        |
| | `search_block` | `search-block.html.twig` | Wrapper du bloc formulaire de recherche                                     |
| | `info_banner_block` | `info_banner-block.html.twig` | Composant bandeau d'information                                             |
| | `header_search_form_block` | `header-search-form-block.html.twig` | Formulaire de recherche modale dans le header                               |
| | `back_to_top_block` | N/A (render array uniquement) | Bouton retour en haut de page                                               |
| | `back_button_block` | `back-button-block.html.twig` | Bouton de navigation retour                                                 |
| **Newsletter** | `newsletter_subscription` | `newsletter-subscription.html.twig` | Page formulaire d'inscription newsletter                                    |
| | `newsletter_confirmation` | `newsletter-confirmation.html.twig` | Page de confirmation post-inscription                                       |
| | `newsletter_unsubscription` | `newsletter-unsubscription.html.twig` | Page formulaire de désinscription                                           |
| | `newsletter_unsubscription_confirmation` | `newsletter-unsubscription-confirmation.html.twig` | Page de confirmation post-désinscription                                    |
| **Erreurs** | `error_403` | `error-403.html.twig` | Page d'erreur accès refusé                                                  |
| | `error_404` | `error-404.html.twig` | Page d'erreur page non trouvée                                              |
| | `error_410` | `error-410.html.twig` | Page d'erreur contenu disparu (film indisponible)                           |

Chaque hook de thème définit un tableau `variables` qui spécifie les données attendues par le template. 

Par exemple, le hook `homepage` attend des variables telles que `header`, `infos`, `collections`, `newsletter` et les blocs de films.

### Définitions des routes

Le module définit des routes personnalisées pour les principales pages de l'application qui ne correspondent pas directement à des nœuds Drupal. 
Les routes sont définies dans le fichier `audiodescription.routing.yml`.

#### Détail des routes

| Nom de route | Chemin | Méthode du contrôleur | Permission | Titre                                              |
|---|---|---|---|----------------------------------------------------|
| `audiodescription.homepage` | `/homepage` | `HomepageController::build` | `access content` | "Accueil"                                           |
| `audiodescription.genres.list` | `/genres` | `GenresController::list` | `access content` | "Films par genre"                                  |
| `audiodescription.movie_search` | `/recherche` | `MovieSearchController::search` | `access content` | "Résultats de recherche"                           |
| `audiodescription.error.404` | `/error-404` | `ErrorController::error404` | `access content` | "Page non trouvée"                                 |
| `audiodescription.error.403` | `/error-403` | `ErrorController::error403` | `access content` | "Accès refusé"                                     |
| `audiodescription.newsletter.subscription` | `/newsletter` | `NewsletterController::subscription` | `access content` | "Les films gratuits de la semaine..."              |
| `audiodescription.newsletter.confirmation` | `/newsletter/confirmation` | `NewsletterController::confirmation` | `access content` | "Votre inscription a bien été prise en compte."    |
| `audiodescription.newsletter.unsubscription` | `/newsletter/desinscription` | `NewsletterController::unsubscription` | `access content` | "Désinscription de la liste de diffusion..."       |
| `audiodescription.newsletter.unsubscription.confirmation` | `/newsletter/desinscription/confirmation` | `NewsletterController::unsubscriptionConfirmation` | `access content` | "Votre désinscription a bien été prise en compte." |
| `audiodescription.error_410` | `/system/410` | `Error410Controller::show410` | `TRUE` | "Contenu indisponible"                             |

Toutes les routes sauf celle de l'erreur 410 requièrent la permission `access content`. La route d'erreur 410 utilise `_access: 'TRUE'` pour être accessible sans authentification, car elle gère les cas où des nœuds films sont dépubliés.

### Intégration au conteneur de services

Le module définit les services personnalisés via `audiodescription.services.yml`, organisés en groupes fonctionnels.

#### Structure des définitions de services

Les services sont définis avec des dépendances de constructeur explicites, suivant le pattern d'injection de dépendances de Drupal. 
La section `_defaults` active l'autoconfiguration.

Patterns clés des services :

- **Entity Managers** : dépendent de `entity_type.manager` et des managers associés pour les opérations CRUD sur les entités
- **Importers** : dépendent des entity managers et de `config_pages.loader` pour la synchronisation des données
- **Breadcrumb Builders** : tagués avec `breadcrumb_builder` et assignés à des valeurs de priorité
- **Event Subscribers** : tagués avec `event_subscriber` pour s'enregistrer auprès du dispatcher d'événements
- **Extensions Twig** : taguées avec `twig.extension` pour enregistrer des filtres personnalisés

### Contrôle de visibilité des blocs

Le module implémente `hook_block_view_alter()` pour contrôler la visibilité des blocs sur des routes spécifiques et gérer le cache du fil d'Ariane.

#### Masquage des blocs sur les pages d'erreur

Sur les pages d'erreur (`audiodescription.error.403` et `audiodescription.error.404`), certains blocs sont masqués pour fournir des mises en page d'erreur plus épurées. Lorsque la route courante correspond à une route d'erreur et que le bloc est dans la liste des blocs masqués, le tableau build du bloc est vidé, empêchant son rendu.


#### Gestion du cache du fil d'Ariane

Le bloc fil d'Ariane a son cache désactivé pour garantir que les fils d'Ariane contextuels sont toujours à jour. 
Cela assure que les sept builders de fil d'Ariane spécialisés (documentés dans [Navigation & Fil d'Ariane (3.6)](https://deepwiki.com/betagouv/audiodescription/3.6-navigation-and-breadcrumbs)) peuvent fournir un contexte de navigation précis en fonction de la route courante et des paramètres de la requête.

De plus, `audiodescription_preprocess_breadcrumb()` définit `max-age` à 0 lors du preprocessing du fil d'Ariane, renforçant la politique de non-cache.

Les blocs de titre de page sont également contrôlés via la configuration des blocs plutôt que par du code.
Par exemple, `drupal/config/sync/block.block.ad_theme_page_title.yml` utilise des paramètres de visibilité pour masquer le titre de page sur les routes `/homepage` et `/genres`, où les mises en page personnalisées gèrent les titres directement.

## Hooks de gestion du cache

Le module implémente des hooks de mise à jour d'entités pour gérer l'invalidation du cache lors des modifications de contenu.

#### Invalidation du cache des collections

Lorsqu'un nœud film est mis à jour, les termes de taxonomie "collection" qui le référencent doivent avoir leur cache invalidé. Cela est géré par `audiodescription_node_update()` :

1. Vérifie si l'entité mise à jour est un nœud `movie`
2. Interroge les termes de taxonomie `collection` qui référencent le film via `field_taxo_movies`
3. Construit le tableau de cache tags au format `taxonomy_term:{$term_id}`
4. Invalide les tags via le service `cache_tags.invalidator`

Cela garantit que lorsque la disponibilité ou les métadonnées d'un film changent, les pages de collections affichant ce film sont rafraîchies au prochain chargement de page.

### Fonctions de preprocessing

Le module fournit des fonctions de preprocessing pour des types de contenu spécifiques afin d'attacher des bibliothèques supplémentaires ou de modifier les render arrays.

#### Preprocessing des nœuds de type page

La fonction `audiodescription_preprocess_node__page()` attache la bibliothèque `iframe_responsive` à tous les nœuds de type page. 
Cela garantit que les iframes intégrées dans le contenu des pages (via WYSIWYG) sont rendues responsives grâce à du CSS/JS personnalisé, améliorant l'affichage sur mobile.

## Architecture de gestion des erreurs

Le module implémente un système sophistiqué de gestion des erreurs pour les films indisponibles, utilisant les codes de statut HTTP 410 (Gone). 

Ce système comprend plusieurs composants travaillant ensemble :

1. **MovieExceptionSubscriber** — intercepte les `AccessDeniedHttpException` pour les nœuds films dépubliés et les convertit en réponses 410
2. **Error410ResponseSubscriber** — gère les erreurs 410 génériques déjà levées sous forme de `HttpException` avec le statut 410
3. **Error410Controller** — fournit une route directe pour les pages 410 lorsqu'elles sont accédées en dehors du flux d'exception

Les trois composants restituent le même template `error_410`, qui affiche un message clair expliquant que le film n'est plus disponible. 
Le template se trouve dans `ad_theme/templates/error/error-410.html.twig`.

La documentation détaillée de ce système est couverte dans [Gestion des erreurs (3.1.3)](https://deepwiki.com/betagouv/audiodescription/3.1.3-error-handling).
