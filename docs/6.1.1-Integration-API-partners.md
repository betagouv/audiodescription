# Intégrations des APIs partenaires

## Objectif et périmètre

Ce document décrit l'implémentation technique des intégrations APIs partenaires dans le backend Patrimony. Chaque partenaire fournit des données de catalogue de films via des mécanismes différents (APIs REST ou fichiers CSV) avec des exigences d'authentification et des formats de données variés. 

Le pipeline d'import transforme ces données hétérogènes en un modèle unifié `SourceMovie` et `Solution`.

Pour l'architecture globale du pipeline d'import et sa planification, voir [Pipeline d'import des données](./6.1-Pipeline-Import.md). 
Pour la logique de transformation et de correspondance des données, voir [Transformation et correspondance des données](6.1.2-Data-Transformation-Matching.md). 
Pour le modèle de données Patrimony, voir [Modèle de données](6.2.0-Data-Model.md).

## Pattern ImporterFactory

Tous les importers partenaires implémentent l'interface `MovieImporterInterface` et sont instanciés via l'`ImporterFactory` selon le pattern factory. 
Cela permet au système de sélectionner l'importer approprié en fonction de la valeur de l'enum `ImportSourceType`.

### Implémentation de la factory

La factory résout les instances d'importers via l'injection de dépendances et retourne l'implémentation appropriée via un `switch` sur les valeurs de l'enum :

- `ImportSourceType::ARTE_TV_API` → `ArteTvApiImporter`
- `ImportSourceType::CANAL_REPLAY_API` → `CanalReplayApiImporter`
- `ImportSourceType::CANAL_VOD_API` → `CanalVodApiImporter`
- `ImportSourceType::FRANCE_TV_API` → `FranceTvApiImporter`
- `ImportSourceType::LACINETEK_API` → `LaCinetekApiImporter`
- `ImportSourceType::ORANGE_VOD_CSV` → `OrangeVodCsvImporter`
- `ImportSourceType::TF1_API` → `Tf1ApiImporter`

## Dépendances aux Entity Managers

Tous les importers reçoivent les dépendances suivantes via l'injection de constructeur :

| Service | Type | Rôle |
|---|---|---|
| `$entityManager` | `EntityManagerInterface` | Persistance des entités Doctrine ORM |
| `$sourceMovieManager` | `SourceMovieManager` | Créer/trouver les entités source movie |
| `$solutionManager` | `SolutionManager` | Créer/mettre à jour les entités solution |
| `$movieManager` | `MovieManager` | Créer les entités film |
| `$httpClient` | `HttpClientInterface` | Requêtes HTTP vers les APIs partenaires |
| `$parameterBag` | `ParameterBagInterface` | Accès aux paramètres de configuration (URLs, tokens) |
| `$solutionRepository` | `SolutionRepository` | Interroger et nettoyer les solutions existantes |
| `$sourceMovieRepository` | `SourceMovieRepository` | Interroger et nettoyer les source movies existants |
| `$movieRepository` | `MovieRepository` | Rechercher les films par IDs externes |
| `$movieFetcher` | `MovieFetcher` | Orchestrer la correspondance film via `findByIds()` |


## Gestion de la configuration

Toutes les configurations d'APIs partenaires sont centralisées dans le fichier de configuration YAML de Patrimony. 
Les variables d'environnement fournissent les valeurs réelles, permettant des configurations différentes selon l'environnement (développement, staging, production). 
Les importers accèdent aux paramètres via le `ParameterBagInterface` injecté.

## Exécution quotidienne des imports

Tous les imports partenaires sont exécutés quotidiennement via cron à **01h05**. 
La cible Makefile `cron-import` exécute tous les imports séquentiellement, avec tolérance aux erreurs (poursuite en cas d'échec).

**Ordre d'exécution :**

1. Canal VOD API
2. Canal Replay API
3. LaCinetek API
4. Arte TV API
5. TF1 API
6. France TV API
7. Import et indexation Drupal

Chaque commande inclut l'option `--create-movies=true` pour déclencher la correspondance et la canonicalisation des films.
