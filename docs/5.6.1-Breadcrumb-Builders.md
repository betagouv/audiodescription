# Fil d'ariane

Ce document décrit les 7 builders spécialisés de fil d'Ariane dans le frontend Drupal, qui fournissent des fils de navigation contextuels pour différents types de pages. 
Chaque builder implémente une logique spécifique à sa route pour construire les hiérarchies de fil d'ariane appropriées selon le contexte de la page courante. 
Ces builders gèrent les pages de détail de films, les résultats de recherche, les termes de taxonomie (genres, réalisateurs, collections), les pages de contenu génériques et les affichages de vues.

Pour les informations sur les patterns de navigation header et footer, voir [En-tête et pied de page (header & footer)](5.6.2-Header-Footer.md).

## Système de correspondance de routes

La méthode `applies()` de chaque builder détermine s'il doit gérer la construction du fil d'Ariane pour la route courante. Le système évalue les builders dans l'ordre d'enregistrement des services jusqu'à ce que l'un retourne `true`.

| Builder | Pattern de route | Type d'entité | Vérification bundle/type |
|---|---|---|---|
| `MovieBreadcrumbBuilder` | `entity.node.canonical` | `node` | `type === 'movie'` |
| `SearchBreadcrumbBuilder` | `audiodescription.movie_search` | N/A | Correspondance exacte du nom de route |
| `CollectionBreadcrumbBuilder` | `entity.taxonomy_term.canonical` | `taxonomy_term` | `bundle() === 'collection'` |
| `NodeBreadcrumbBuilder` | `entity.node.canonical` | `node` | `type === 'article'` OU `'page'` |
| `DirectorBreadcrumbBuilder` | `entity.taxonomy_term.canonical` | `taxonomy_term` | `bundle() === 'director'` |
| `GenreBreadcrumbBuilder` | `entity.taxonomy_term.canonical` | `taxonomy_term` | `bundle() === 'genre'` |
| `ViewBreadcrumbBuilder` | Toute route | N/A | Possède les paramètres `view_id` et `display_id` |

## MovieBreadcrumbBuilder

Construit les fils d'ariane pour les pages de détail des nœuds films, en créant un chemin à travers la hiérarchie des genres.

**Structure du fil d'ariane :**

```
Accueil > [Titre de la vue Genres] > [Vue de recherche filtrée sur le 1er genre] > [Titre du film]
```

**Fonctionnalités clés :**

- Récupère le premier genre depuis le champ de référence `field_genres`
- Charge dynamiquement le titre de la vue "genres" via `Views::getView()`
- Crée un lien de recherche intermédiaire avec le paramètre de filtre genre
- Désactive le cache pour garantir les mises à jour du contenu dynamique

## SearchBreadcrumbBuilder

Fournit un fil d'Ariane simple pour la page de résultats de recherche de films.

**Structure du fil d'Ariane :**

```
Accueil > Résultats de recherche
```

**Implémentation :**

- S'applique uniquement à la route `audiodescription.movie_search`
- Crée un fil d'Ariane à 2 niveaux avec lien d'accueil et label "Résultats de recherche"
- Désactive le cache avec `setCacheMaxAge(0)`


## CollectionBreadcrumbBuilder

Gère les fils d'ariane pour les pages de termes de taxonomie Collection.

**Structure du fil d'Ariane :**

```
Accueil > [Nom de la collection]
```

**Implémentation :**

- S'applique à la route `entity.taxonomy_term.canonical` avec `bundle() === 'collection'`
- Ajoute le contexte de cache `url` pour la prise en compte des paramètres de requête
- Crée un lien vers l'URL canonique de la collection
- Désactive le cache avec `setCacheMaxAge(0)`

## NodeBreadcrumbBuilder

Construit les fils d'ariane pour les nœuds article et page, en suivant la hiérarchie du menu si disponible.

**Structure avec menu :**

```
Accueil > [Chemin du menu] > [Titre de la page courante]
```

**Structure sans menu (repli) :**

```
Accueil > [Titre du nœud]
```

**Fonctionnalités clés :**

- Injection de dépendances : `EntityTypeManagerInterface` et `MenuLinkTreeInterface`
- Parcours du chemin actif du menu principal pour construire la hiérarchie
- Extraction de l'UUID depuis les identifiants des liens de menu (suppression du préfixe `menu_link_content:`)
- Repli sur un lien simple avec le titre du nœud si aucun chemin de menu n'existe
- Ajout des contextes de cache `url.path` et `url`

## DirectorBreadcrumbBuilder

Fil d'ariane simple pour les pages de termes de taxonomie Réalisateur.

**Structure du fil d'Ariane :**

```
Accueil > [Nom du réalisateur]
```

**Implémentation :**

- S'applique à la route `entity.taxonomy_term.canonical` avec `bundle() === 'director'`
- Ajoute le contexte de cache `url`
- **Ne désactive pas** le cache (pas d'appel `setCacheMaxAge(0)`)

## GenreBreadcrumbBuilder

Construit les fils d'ariane pour les pages de termes de taxonomie Genre, avec un lien intermédiaire vers la liste des genres.

**Structure du fil d'Ariane :**

```
Accueil > [Titre de la vue Genres] > [Nom du genre]
```

**Implémentation :**

- Charge la vue "genres" pour récupérer son titre dynamique via `Views::getView()`
- Crée un lien vers la route `view.genres.page`
- Ajoute un lien vers la page du terme de taxonomie genre
- Ajoute le contexte de cache `url`


## ViewBreadcrumbBuilder

Fil d'ariane générique pour les pages de vues, utilisant la hiérarchie de menu ou le titre de la vue.

**Structure avec menu :**

```
Accueil > [Chemin du menu] > [Titre de la vue]
```

**Structure sans menu (repli) :**

```
Accueil > [Titre de la vue]
```

**Fonctionnalités clés :**

- Injection de dépendances identique au `NodeBreadcrumbBuilder`
- Détecte les routes de vues en vérifiant la présence des paramètres `view_id` et `display_id`
- Repli sur l'exécution de la vue pour récupérer le titre quand aucun chemin de menu n'existe
- Ne désactive pas le cache (s'appuie sur les contextes de cache par défaut)

## Stratégie de cache

Les builders implémentent différentes stratégies de cache selon la volatilité du contenu :

* `setCacheMaxAge(0)` (pas de cache) lorsque le contenu change fréquemment ou qu'il existe des relations dynamiques
* `addCacheContexts(['url'])` lorsque les paramètres d'URL affectent la structure du fil d'Ariane
* `addCacheContexts(['url.path'])` lorsque la construction du chemin de menu dépendante du path

**Pattern de désactivation du cache :**

Utilisé dans 5 builders pour forcer la regénération à chaque requête. 

```php
// Désactiver totalement le cache en supprimant toutes les métadonnées.
$breadcrumb->addCacheableDependency((new CacheableMetadata())->setCacheMaxAge(0));
```

## Patterns communs

### Pattern : Ajout du lien vers la page d'accueil

Tous les builders démarrent avec le même lien vers la page d'accueil :

```php
Link::createFromRoute(t('Home'), '<front>')
```

### Pattern 2 : Extraction des paramètres de route

Récupération d'un terme de taxonomie :

```php
$term = $route_match->getParameter('taxonomy_term');
```

Récupération d'un nœud :

```php
$node = $route_match->getParameter('node');
```

### Pattern 3 : Chargement du titre d'une vue

Utilisé par `MovieBreadcrumbBuilder`, `GenreBreadcrumbBuilder` et `ViewBreadcrumbBuilder` :

```php
$view = Views::getView('genres');
$view->setDisplay('page');
$title = $view->getTitle();
```

### Pattern 4 : Parcours du menu

Utilisé par `NodeBreadcrumbBuilder` et `ViewBreadcrumbBuilder` : 
1. Chargement de l'arbre du menu principal
2. Itération en sens inverse sur le chemin actif
3. Extraction de l'UUID depuis `menu_link_content:{uuid}`
4. Chargement de l'entité
5. Création du lien.

## Enregistrement des services

Tous les builders de fil d'ariane sont enregistrés dans `audiodescription.services.yml` avec le tag `breadcrumb_builder` et des valeurs de priorité appropriées pour contrôler l'ordre d'évaluation.

Les valeurs de priorité plus élevées sont évaluées en premier. 
Les builders movie et search ont typiquement des priorités plus élevées pour s'assurer qu'ils correspondent avant les builders plus génériques comme le `ViewBreadcrumbBuilder`.
